# 色谱峰处理算法需求梳理（评审稿 v1）

> 本稿仅完成需求文档和样例数据的梳理，尚未开始 GitHub 开源算法检索与试跑。待需求口径确认后进入候选算法筛选、复现和对比测试。

## 1. 项目目标

输入一维色谱时间序列，先识别基线状态和异常干扰，再完成正/负峰检测、峰起止点和峰顶定位、重叠峰拆分、组分定性、峰高/峰面积定量，并输出异常与结果可信度提示。

目标组分为 H2、CO、CH4、CO2、C2H4、C2H6、C2H2。算法除正常谱图外，需重点处理异常噪声、漂移/鼓包、毛刺、正负峰重叠和保留时间漂移等情况。

## 2. 文档中的问题口径

需求文档正文实际列出 **6 类异常问题**，而非 5 类，且编号从“谱图类型5”跳到“谱图类型7”：

1. 动态噪声识别：不同设备/工况噪声水平不同；不能依赖固定阈值，要动态估计局部噪声、自适应滤波、动态跟踪 SNR，并判定峰起止点；过大基线噪声还要给出仪器异常提示。
2. 基线漂移识别与修正：识别温度或流速变化造成的单向、近线性漂移；漂移下仍要检出小峰；校正后控制峰高和峰面积误差。
3. 平缓峰/基线波动与真峰区分：超出正常范围的宽峰或缓慢波动不能按组分峰处理；综合峰宽、对称性、斜率等特征判别。
4. 重叠峰及“大峰上出小峰”：识别连续重叠峰各自峰顶；从缓慢鼓包、大峰或拖尾上识别小的有效峰，并分别定量。
5. 组分峰与毛刺干扰区分：综合峰宽、对称性、上升/下降斜率等特征排除极窄随机脉冲，即使毛刺与目标保留时间重合也不能误报。
6. 负峰识别：识别独立负峰，以及负峰在正峰前、后或中间的重叠；不能把“负峰 + 正峰”错误拆成两个正峰；空气负峰存在时仍需正确识别和定量邻近 H2 峰。

需请需求方确认：所谓“5 个问题”是否是将第 2、3、4 项统一归入“基线/鼓包问题”，或负峰不在本轮五项范围内；否则后续应按六项测试。

## 3. 不应遗漏的横向能力

- 基线状态先验分类：正常、漂移、鼓包、大噪声、毛刺、负峰，可多标签共存；随后选择相应处理策略。
- 多参数联合判别：保留时间、峰宽/FWHM、峰形对称性或偏度、上下坡斜率、峰高、峰面积、局部 SNR、峰类型。
- 保留时间自动跟踪：以 H2、CO 等基准峰为锚点，对同通道所有组分的预期保留时间整体校正；允许 ±5% 漂移。
- 高动态范围：同图最大峰和最小峰峰高相差 1000 倍时仍识别全部目标峰。
- 过载/非理想峰形：前伸、拖尾等高浓度峰仍需正确定性和定量。
- 质量控制：对高噪声、信号中断、超量程、不可可靠定量等情况输出明确告警，不能只强行给峰结果。

## 4. 建议的处理流水线

1. **输入与质量检查**：验证时间单调性、采样间隔、缺失值、饱和/中断、通道和单位；统一数据结构。
2. **毛刺预检测**：用局部中位数残差、脉冲宽度、斜率反转和邻域一致性标记瞬态毛刺；保留掩码，避免滤波把毛刺扩散成假峰。
3. **基线与局部噪声联合估计**：估计缓慢基线和随时间变化的稳健噪声尺度。基线模型应能覆盖线性漂移和鼓包，但不能吞掉真实宽峰或大峰拖尾上的小峰。
4. **自适应降噪**：滤波强度随局部噪声变化；同时约束峰高、峰宽、面积和峰顶位置的失真。
5. **多尺度候选峰检测**：同时检测正峰和负峰；在多个尺度寻找峰顶/肩峰，避免单一导数阈值漏掉大小差异很大的峰。
6. **峰起止点判定**：结合校正后信号、局部噪声、导数、峰谷和最小持续宽度定位起止点，不采用全图固定阈值。
7. **重叠峰拆分**：先用肩峰/曲率/多尺度响应估计子峰个数和初值，再进行受约束峰形拟合或局部解卷积；基线项与子峰项应联合优化。
8. **特征提取与真假峰判别**：提取宽度、半峰宽、对称性/偏度、上升下降斜率比、面积、高度、SNR、尖锐度、残差等；规则模型优先，必要时再评估轻量机器学习模型。
9. **保留时间对齐与组分定性**：用基准谱图建立各设备/通道模板，通过锚点峰拟合整体时间映射；候选峰需同时满足时间、形状和 SNR 等条件。
10. **定量与质量标记**：在校正基线上积分，输出面积、高度、误差/可信度以及噪声过大、毛刺、重叠、负峰、漂移等标记。

建议算法输出至少包括：通道、组分、峰类型、起点/峰顶/终点、保留时间、峰高、面积、峰宽、FWHM、局部 SNR、基线值、置信/质量标记；另输出校正基线、去噪曲线和异常区间，方便逐点复核。

## 5. 量化验收指标

| 项目 | 文档指标 |
|---|---|
| 痕量峰检出 | SNR >= 3 的目标峰检出率 >= 99%；C2H2 为 100% |
| 误报 | 整体定性误报率 <= 0.1%；C2H2 零误报 |
| 漏检 | 7 种目标组分在浓度不低于定量下限时，整体漏检率 <= 0.5% |
| 峰边界 | 起点、落点位置误差不超过该峰半峰宽的 2% |
| 漂移下定量 | 线性漂移不超过满量程 10% 时，面积相对误差 <= 3% |
| 重叠峰 | 峰谷比 >= 10% 时正确拆分，单峰面积误差 <= 5%；大峰拖尾上小峰高度 >= 大峰 5% 时必须识别 |
| 保留时间漂移 | 同通道整体漂移 ±5% 内，所有组分定性准确率 100% |
| 毛刺排除 | 正确排除率 >= 99.9%，不得误判为组分峰 |
| 负峰 | 空气负峰存在时 H2 定性 100%，面积误差 <= 5% |
| 重现性 | 连续 7 次：保留时间 RSD <= 0.5%，面积 RSD <= 1% |
| 异常样例 | 所有已标注异常谱图通过率 100% |
| 性能 | 50 Hz、4 小时、72 万点；RK3568 Cortex-A35 1.8 GHz 上处理时间 <= 15 秒 |

最终还需在不少于 1000 张的盲测集上与现有算法比较：异常工况显著更优，正常样品结果一致性 >= 99%，重现性不劣化。

## 6. 运行与交付约束

> 2026-07-15 评审口径更新：本轮开源算法检索与 Python 试跑暂不以 C++/ARM 可移植性为筛选条件；以下内容仍是原需求文档中的最终交付约束，留待算法效果确认后再评估。

- 最终实现：C++11，GCC 7.5+；Windows/Linux x86_64 和 RK3568 ARM Linux。
- 设备：ARM Cortex-A35 1.8 GHz，2 GB 内存；前期允许 Python 验证，最终转 C++。
- 第三方库应尽量少；必须提供源码和许可证，并能在 RK3568 静态链接，无运行时依赖。
- 如使用 Qt，版本不高于 5.12。
- 交付源码、设计文档、三个平台的构建/集成说明、测试报告、带标注测试集及默认参数。
- 若采用机器学习，还需交付模型、训练代码、标注规范和重训练方法。

最终工程化阶段不能只看“效果图”：许可证、C/C++ 可移植性、依赖规模、确定性、复杂度、ARM 静态构建能力和可二次开发性仍需另行评估。

## 7. 当前样例数据盘点

- 共 45 个 CSV，均为两列：`x,Curvel`，无缺失/非数值行。
- `x` 以分钟计，间隔约 0.0016667 分钟，即 0.1 秒/点、10 Hz；单文件 3221～7741 点，约 5.37～12.9 分钟。
- 五个目录：基准谱图 5 个、鼓包 13 个、噪音 14 个、负峰 7 个、电信号干扰 6 个。
- 与最终性能条件不同：样例为 10 Hz 短序列，生产性能指标是 50 Hz、4 小时、72 万点。
- CSV 只有原始曲线，没有组分、峰顶、峰起止点、基线、峰面积或应排除区间等逐峰真值；文件夹/文件名只提供弱标签。
- 存在 4 组完全重复文件，实际独立曲线为 41 条：
  - 电信号干扰 `1211122026-1-5_14_13_40.CSV` = 鼓包 `A13-C2H2赶上鼓包.CSV`
  - 电信号干扰 `2026-1-5_14_13_40.CSV` = 电信号干扰 `H4-通道1感觉是信号中断采集.CSV`
  - 噪音 `B5-噪音大-通道1.CSV` = 噪音 `噪音多且通道2没有峰信息.CSV`
  - 负峰 `2026-1-20_11_46_56.CSV` = 负峰 `CO2负峰导致C2H4不组分识别2026-1-21_16_59_52.CSV`

数据目录与文档六类问题不是一一对应：没有独立的“线性基线漂移”和“平缓峰”目录；“鼓包”可能同时覆盖平缓基线与大峰上小峰。需要给每条曲线补充多标签场景定义。

## 8. 在证明“跑出正确结果”前必须补齐的信息

### 最小必需标注

每个 CSV 至少应给出：

- 设备/机型、通道、单位、采样率、样品/标气类别；
- 每个真实峰的组分、起点、峰顶、终点、正/负峰类型；
- 参考基线或确认过的峰高、峰面积；
- 应排除的噪声、鼓包、毛刺、信号中断等区间；
- 哪些小峰是“必须检出”，哪些起伏是“必须排除”；
- 对重叠峰给出子峰数及各子峰参考面积；
- 标准样/同一样品重复进样之间的对应关系。

### 需明确的定义

- SNR 的噪声窗口、噪声统计量和计算公式；
- 峰起止点的人工标注规则，以及“半峰宽 2%”误差的具体分母；
- 峰谷比和“满量程 10%”的计算方法；
- 定量下限、各组分/通道的正常峰宽范围及保留时间窗口；
- 面积误差的参考真值来自人工积分、标气浓度还是现有算法；
- 毛刺“不得误判”与总体毛刺排除率的样本统计单位；
- C2H2 零误报是针对本次盲测集，还是产品全生命周期要求。

没有上述真值时，下一阶段仍可把开源算法跑在 45 个样例上并画出基线、候选峰和排除区间供人工评审，但只能称为“候选效果展示”，不能声称已达到文档的准确率和定量指标。

## 9. 确认后下一阶段工作

1. 按模块搜索开源实现，而不是寻找一个包办全部问题的仓库：基线校正、稳健降噪/毛刺抑制、多尺度峰检测、重叠峰拟合、保留时间对齐、峰形特征与分类。
2. 优先筛选宽松许可证、源码完整、可离线、可 C++11/RK3568 移植、依赖少且有测试或论文依据的候选；记录仓库活跃度、许可证和移植成本。
3. 用统一适配层在 41 条独立曲线上运行候选组合，输出叠图和逐峰表；重复曲线只用于一致性检查，不跨训练/测试集。
4. 有真值时计算检出、误报、边界、面积、保留时间和运行时指标；无真值时先生成待人工确认的预标注结果。
5. 根据结果确定“经典信号处理主链 + 少量规则/轻量分类器”是否足够，再决定是否引入机器学习；工程约束下不建议一开始就依赖重型深度学习框架。
