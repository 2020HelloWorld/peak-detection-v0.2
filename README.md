# ChromPeak 色谱峰检测工程 v0.4.1

这是一个针对一维色谱时间序列的 Python 验证工程。当前目标是先可靠完成预处理、峰位置检测和峰形类型判定，再利用人工真值迭代参数；暂不处理 C++/ARM 移植。

## 当前能力

- 动态噪声：局部一阶差分 MAD、局部 SNR、高噪声工况保护。
- 基线漂移：arPLS、双极性稳健基线及 rolling-ball 局部背景。
- 正峰与负峰：独立检测分支，达到统一置信度阈值的负峰可直接确认。
- 宽峰与鼓包：区分普通正峰、宽峰、鼓包背景上的峰及宽背景候选。
- 平缓峰：新增保守的平缓宽峰候选规则，单独标记但仍保留人工复核属性。
- 重叠峰：根据峰窗口交叠和模板宽度输出重叠/未分离候选。
- 电信号干扰：综合脉冲、宽度、对称性和峰顶平坦度判定。
- 结果解释：逐峰输出分项得分、综合峰置信度和模板匹配置信度；密集区域的文字标签会自动二维避让；确认峰在图中另标算法起止点和时间。

需求文档中的 5.1～5.6 六类问题、验收指标和数据盘点见 [需求梳理](docs/需求梳理_v1.md)。

## 完整算法流程

当前程序的实际处理主链如下。它是“信号处理 + 规则分类 + 参考模板”的组合算法，不是端到端神经网络：

```text
ZIP/CSV 原始数据
  → 格式校验与 SHA-256 精确去重
  → 从基准谱学习 T1～T6 保留时间/峰宽模板
  → 每条曲线进行孤立脉冲识别、基线估计和动态噪声估计
  → 构造正峰检测分支与保留正负号的检测分支
  → 正、负候选峰分别搜索
  → 计算峰顶、边界、宽度、突出度、SNR、面积和形态特征
  → 匹配参考模板并计算两类置信度
  → 判定普通峰、负峰、宽峰、重叠峰、鼓包、电干扰或噪声候选
  → 按 confirmed/review/artifact 分级
  → 输出 CSV、统计表和三层对照图
```

### 阶段 1：原始数据读取、校验与去重

1. 从 ZIP 中读取所有 `.CSV`，要求列名严格为 `x,Curvel`。`x` 是时间轴，`Curvel` 是原始检测器响应。
2. 将两列转换为浮点数，检查是否存在 `NaN/Inf`、采样点是否少于 5 个、时间轴是否严格递增。任一检查失败会直接报错，不会静默修补原文件。
3. 对 CSV 原始字节计算 SHA-256。内容完全相同的文件只处理一次，同时在 `manifest.json` 中保留重复文件分组。
4. `Curve.y` 始终保存原始 `Curvel` 数值。结果图第一层直接绘制 `Curve.x/Curve.y`，不会被后续预处理覆盖。

相关技术：Pandas CSV 解析、NumPy 数值校验、SHA-256 内容哈希。

### 阶段 2：从基准谱学习 T1～T6 模板

程序把目录名以 `1-` 开头的 5 条基准谱作为参考数据。基准谱先经过与普通数据相同的预处理和正峰候选检测，再保留满足以下条件的稳定候选：

- 局部 `SNR ≥ 8`；
- 相对突出度 `≥ 0.003`；
- 95% 峰宽位于 `0.025～0.80 min`。

随后只按峰顶保留时间进行 complete-linkage 层次聚类，聚类距离阈值为 `0.120 min`。一个聚类至少要得到 4 条不同基准谱支持，才会成为稳定模板。对每个模板统计：

- 保留时间中位数；
- 自适应保留时间容差，限制在 `0.045～0.22 min`；
- 峰宽中位数、允许的峰宽下限和上限；
- 对称性中位数和允许下限；
- 支持该模板的基准谱数量。

模板按保留时间排序后命名为 T1、T2……。T1～T6 只代表保留时间模板槽，不自动代表 H2、CO、CH4 等化学名称；化学名称必须通过 `configs/component_map.json` 配置。

相关技术：参考样本筛选、SciPy complete-linkage 层次聚类、中位数与范围稳健统计。

### 阶段 3：单条谱图预处理

#### 3.1 采样间隔和有效量程

采样间隔 `dt` 取相邻时间差的中位数。用于归一化的稳健信号量程定义为原始信号 `0.1%` 与 `99.9%` 分位数之差，避免单个极端值完全决定量程。

#### 3.2 仅识别孤立单点脉冲

对原始信号使用 3 点中值滤波得到局部参考，计算原始值与局部中值之差。脉冲阈值取下面两者的较大值：

- 残差稳健标准差的 8 倍；
- 整条曲线稳健量程的 `0.4%`。

超过阈值的孤立点只在正峰预处理分支中用 3 点中值替换。连续较宽的事件不会在此阶段删除，而是保留到后面的峰/干扰分类。原始数据数组和原始谱线图不做替换。

相关技术：3 点中值滤波、MAD 稳健标准差、脉冲掩码。

#### 3.3 三种基线/背景估计

程序同时计算三类背景信息：

1. **双极性稳健基线**：约 `1.0 min` 的中值滤波，再用约 `0.35 min` 的二阶 Savitzky–Golay 趋势平滑。它不优先偏向正峰或负峰，是负峰分支使用的基线。
2. **arPLS 基线**：`pybaselines.arpls`，当前 `λ=1e7`、最多 50 次迭代，用于跟随普通慢漂移。
3. **rolling-ball 局部背景**：约 `0.50 min` 的半窗口和约 `0.025 min` 的平滑半窗口，用于描述鼓包和局部背景变化。

强负峰可能把下包络基线长时间向下拉，因此正峰主基线取 `max(arPLS, 双极性基线)`；局部鼓包背景取 `max(rolling-ball, 双极性基线)`。

相关技术：中值趋势基线、Savitzky–Golay、arPLS 非对称重加权最小二乘、rolling-ball 背景估计。

#### 3.4 构造检测分支

- **正峰分支**：`孤立脉冲清理后的信号 - 正峰主基线`，再进行约 `0.015 min` 窗口、三阶多项式的 Savitzky–Golay 轻度平滑。
- **鼓包辅助分支**：`孤立脉冲清理后的信号 - rolling-ball 局部背景`，用于局部背景证据。
- **带符号分支**：`原始信号 - 双极性基线`，再轻度平滑。该分支保留正负方向，负峰搜索使用其相反数。

这些分支是供检测使用的基线校正信号，不是高斯、洛伦兹或 EMG 峰函数拟合结果。

#### 3.5 随时间变化的噪声估计

每个采样点的局部噪声取以下两种估计的较大值：

1. 短窗平滑残差在约 `0.40 min` 窗口内的局部 MAD；
2. 信号一阶差分在约 `0.60 min` 窗口内的局部 MAD。

因此 SNR 使用的是随时间变化的局部噪声，而不是整条曲线共用一个固定标准差。文件级噪声指标为局部噪声中位数除以稳健信号量程。若该指标同时高于基准谱中位噪声的 10 倍和 `5×10⁻⁵`，则标记为高噪声工况。

基线漂移量定义为主基线末端值减去起始值；漂移绝对值达到原始信号全量程的 5% 时，在文件汇总中标记基线漂移。

相关技术：滑动 MAD、稳健一阶差分噪声、局部 SNR、参考噪声归一化。

### 阶段 4：正峰和负峰候选检测

正峰使用蓝色正峰分支；负峰使用带符号分支的相反数。二者分别调用 SciPy `find_peaks`，当前最低候选条件为：

| 条件 | 当前规则 |
|---|---|
| 峰高 | `max(2 × 局部噪声, 0.0005 × 稳健信号量程)` |
| 突出度 | `max(3 × 局部噪声, 0.0005 × 稳健信号量程)` |
| 最小宽度 | 3 个采样点 |
| 最小峰间距离 | `max(3个采样点, 0.008 min)` |
| 突出度搜索窗口 | 最长约 `2.0 min` |

这意味着算法扫描完整时间范围，但不会把每一个局部极值都输出为候选。低于局部噪声门槛的微小起伏会在这一阶段被忽略。

相关技术：SciPy `find_peaks`、动态高度/突出度数组、正负双分支搜索。

### 阶段 5：逐候选特征计算

对每个候选计算下列特征：

| 特征 | 计算含义 |
|---|---|
| `apex_time_min` | 候选峰顶对应的原始时间坐标。 |
| `start/end_time_min` | SciPy 突出度基准上相对高度 95% 处的近基线边界。它是算法边界，不是已人工确认的积分边界；检测图仅为 `confirmed` 峰显示这两个端点。 |
| `width_min` | 上述 95% 相对高度宽度。 |
| `fwhm_min` | 半高宽。 |
| `top_width_ratio` | 10% 相对高度处宽度与半高宽的比值，用于识别平顶/异常峰顶。 |
| `height/prominence` | 在相应预处理检测分支上的峰高和突出度。 |
| `relative_prominence` | 突出度除以本文件稳健信号量程。 |
| `snr` | 突出度除以峰顶位置的局部噪声。 |
| `area` | 在 95% 边界内，对基线校正检测信号的正值做梯形积分；当前不是最终定量面积。 |
| `symmetry` | 峰顶左右半高宽较小值除以较大值，越接近 1 越对称。 |
| `bilateral_depth` | 对轻度平滑后的原始谱线检查峰顶相对左右肩部是否同时突出/下探；负峰必须形成真实双侧谷形。 |
| `signed_depth_relative` | 带符号分支相对本文件量程的上升或下探深度。 |
| `baseline_change_ratio` | 候选范围内 rolling-ball 局部背景变化量相对峰高的比例。 |
| `impulse_overlap` | 峰顶附近是否与单点脉冲掩码重合。 |
| `edge_distance_min` | 峰顶距采集起点或终点的最小距离。 |

### 阶段 6：模板匹配与两种置信度

候选只在峰顶落入某个模板的保留时间容差时匹配该模板。模板匹配置信度 `template_confidence` 的组成是：保留时间 50%、峰宽 30%、对称性 15%、峰顶非平坦度 5%。它回答“该峰像不像 T1～T6 中的某个模板”。

峰存在置信度 `peak_confidence` 回答“该事件像不像真实色谱峰”。它与模板匹配置信度分开，避免明显峰因为保留时间漂移或暂时没有组分映射而被直接否定。峰宽得分会和所有参考模板宽度比较，取最相似者。

各证据先映射为 0～1 分数。SNR 在约 3～30 之间进行对数映射，相对突出度在正常门槛到 `0.03` 之间进行对数映射；峰宽按与参考宽度的对数比例评分，对称性、非平顶度和负峰下探深度分别归一化。正负峰的详细权重见后面的“统一置信度”。

当前会施加的主要惩罚包括：

- 高噪声且相对突出度不足：峰置信度乘 `0.75`；
- 靠近采集边缘：乘 `0.65`；
- 模板窗口内疑似电干扰窄峰：乘 `0.45`；
- 明显电尖峰：乘 `0.25`；
- 超宽鼓包/基线候选：乘 `0.80`；
- 负峰与脉冲重合：乘 `0.65`；
- 两侧都有正峰的浅负向谷底：乘 `0.65`。

### 阶段 7：峰类型分类

#### 7.1 正峰

落入模板保留时间窗口的候选：

- 峰宽和对称性均在模板范围内：`normal_positive_peak`；局部背景变化比例达到 0.25 时改为 `positive_peak_on_hump`。
- 比模板窄，并且与脉冲重合、峰顶偏平或明显不对称：`electrical_interference_candidate`；否则为 `narrow_positive_peak`。
- 比模板明显宽：`broad_or_overlapped_peak`。
- 其余形态不确定：`uncertain_peak_or_noise`。

不在任何模板窗口的候选：

- 宽度小于 `0.025 min` 或与脉冲重合：`electrical_spike`。
- 宽度大于 `0.80 min` 时先检查平缓宽峰规则；满足规则则为 `gentle_broad_peak_candidate`，否则为 `broad_hump_or_baseline`。普通宽背景若最终置信度达到确认线，再改为 `broad_positive_peak`。
- 峰宽仍像某个参考峰且具有最低对称性：`unassigned_positive_peak`，表示确认有峰但保留时间未映射。
- 其余：`uncertain_peak_or_noise`。

模板窗口外极小的波纹会根据普通/高噪声相对突出度门槛直接过滤，避免输出大量假峰。模板窗口内的小峰仍保留竞争资格，防止弱组分被统一删除。

`gentle_broad_peak_candidate` 的中文标注是“平缓宽峰候选”。为了符合“很大概率”而不是把所有宽事件改名，当前采用高精度、低召回的保守条件：不匹配模板、无单点脉冲重合、95% 峰宽不小于 `0.80 min`、半高宽不小于 `0.30 min`、95% 峰宽与半高宽之比不大于 6、对称性不小于 0.25、峰顶宽度比不大于 0.65、相对突出度不小于 0.003、SNR 不小于 8、局部背景变化比例不大于 3、原始谱线双侧回落深度比例不小于 0.002，并且远离采集边缘。

该类型仍叫“候选”，不能直接解释为化学组分峰。真正的慢波动可能已经被基线算法吸收；要完整区分平缓峰和缓慢基线，还需要在扣基线前增加专门的低频背景检测分支和人工真值验证。

#### 7.2 负峰

负峰首先必须同时满足带符号下探深度和原始谱线双侧谷深门槛；高噪声时两个门槛都会提高。通过后：

- 宽度小于 `0.018 min`：`negative_electrical_spike`；
- 宽度不超过 `0.80 min`：`negative_peak`；
- 更宽：`broad_negative_peak`。

如果负向事件较浅，并且前后 `0.8 min` 范围内都有正峰，则改为 `interpeak_valley_or_negative_peak` 并降低置信度，避免把峰间谷底强行判成化学负峰。

#### 7.3 重叠和同模板多候选

- 同一个模板窗口内可以保留多个确认峰。模板匹配最好的候选保留原类型，其余标记为 `secondary_or_overlapping_candidate`，但不会仅因同槽多峰而撤销其峰存在状态。
- 相邻确认正峰的 95% 峰窗口发生交叠时，两者标记为 `overlapping_positive_peak`。

当前方法只做重叠关系识别，还没有进行多峰参数化解卷积。

#### 7.4 “正向毛刺/电尖峰”和“电信号干扰候选”

`electrical_spike` 现在显示为“正向毛刺/电尖峰”。它指正方向上的极窄瞬态事件，或者峰顶附近和单点脉冲掩码重合的事件；在当前数据中它通常更像毛刺或采集电干扰，而不像正常色谱正峰。

`electrical_interference_candidate` 表示事件可能落在正常保留时间窗口内，但相对模板明显过窄，并伴随单点脉冲、峰顶偏平或不对称等异常形态。它可能不是单点毛刺，而是持续稍长的电信号异常。`negative_electrical_spike` 是相同概念的负方向版本。

这些名称是依据波形形态给出的算法分类，不能仅凭谱线确定硬件物理故障原因。也就是说，“正向毛刺/电尖峰”属于电干扰/伪峰大类，但“电干扰”不一定都表现为单点尖峰。

### 阶段 8：状态分级

峰类型描述“它像什么”，状态描述“当前证据有多可靠”，两者不能混为一谈：

| 状态 | 当前规则 |
|---|---|
| `confirmed` | `peak_confidence ≥ 0.75`，正峰和负峰使用同一确认阈值。 |
| `artifact` | `peak_confidence < 0.45`，且类型属于电信号事件或 `uncertain_peak_or_noise`。 |
| `review` | 其他未达到确认线的事件，包括证据不足、宽峰/谷底歧义或低置信度负峰。 |

因此，`unassigned_positive_peak` 的“未映射”不等于未确认；它只表示尚未匹配 T1～T6/化学组分。`artifact` 表示检测到了信号事件但更像干扰或噪声，不表示程序没有在该位置检测到变化。

### 阶段 9：结果输出与可视化

每条曲线都会生成三层图：未处理原始谱线、基线诊断、预处理检测信号及分类标记。第三层对每个 `confirmed` 峰在算法起点画空心向右三角、在算法终点画空心向左三角，并把 `起 x.xxxx / 止 x.xxxx min` 写入该峰标签；`review`、`artifact` 和 `likely noise` 不画起止端点。标签排版会读取 Matplotlib 实际文字边界框，在像素坐标中尝试上下和左右多个位置，避开其他标签、坐标轴边界和图例，并用细引导线连接峰顶。确认峰标签具有最高优先级并始终保留；密集区域如果确实没有无碰撞位置，只省略低优先级待复核文字。随后按状态拆分 CSV，并生成逐文件噪声/漂移汇总、分类型置信度统计、模板 JSON 和人工真值填写模板。

需要特别注意：当前 `area` 和 `start/end_time_min` 是自动候选阶段的基线校正面积和算法边界，适合对比及人工复核，但还未经过人工积分边界和标准物质标定，不能直接当作最终浓度定量结果。`rel_height=0.95` 表示从峰顶向突出度基准下降 95% 的近基线交点，不是“峰高的 95%”；背景漂移、长尾或重叠时，自动区间可能偏宽，图中显示并不等于边界已经人工确认。

### 当前尚未完成的能力边界

- 未配置 `component_map.json` 前，T1～T6 不能自动解释为具体化学组分。
- 尚未进行高斯/EMG 等参数化峰拟合和重叠峰解卷积。
- 尚未用人工逐峰真值校准置信度，因此它是可解释的综合评分，不是严格统计概率。
- 尚未获得独立盲测集上的检出率、误报率、类型混淆矩阵和边界误差。
- 当前面积不是经过标准曲线、响应因子和人工积分边界校准的定量面积。
- 平缓宽峰目前是保守候选规则，尚未建立扣基线前的多尺度低频检测和真值闭环。

## 如何理解检测结果图

每张检测图现在固定分成三层，三层使用同一个时间轴：

1. **原始数据谱线**：直接读取 CSV 的 `x` 和 `Curvel` 两列并连线。该层不做滤波、平滑、孤立点替换或基线扣除，用于和算法结果直接对照。
2. **基线诊断图**：浅灰色是原始数据参考，红色是稳健主基线，紫色是 rolling-ball 局部背景。这里的红、紫曲线是预处理估计结果，不是原始测量值。
3. **检测信号图**：蓝色 `preprocessed positive` 是孤立脉冲清理、基线扣除和轻度 Savitzky–Golay 平滑后的正峰分支；灰色 `signed branch` 是保留正负号的分支，主要用于负峰。峰顶、类别以及确认峰的起止端点都画在这一层。

第三层不是对每个峰进行高斯、洛伦兹或 EMG 函数拟合后的曲线，而是供候选检测和形态分析使用的预处理信号。峰顶时间仍对应原始时间轴，负峰确认还会额外检查原始数据两侧是否确实形成谷形。

算法会扫描整条时间范围，并分别搜索正峰和负峰，但图中的标记表示“通过最低候选门槛的事件”，不是原始曲线的每一个局部极值。明显低于局部噪声门槛、突出度不足或持续时间不足三个采样点的细小波动不会进入候选集合。所有保留下来的候选都记录在 `all_detected_features.csv` 中。

图例符号含义：

| 符号 | 图例 | 含义 |
|---|---|---|
| 绿色圆点 | `confirmed` | 综合峰置信度达到确认阈值，当前默认阈值为 0.75。 |
| 橙色三角 | `review` | 有峰证据，但置信度不足或形态存在歧义，需要人工复核。 |
| 灰色菱形 | `likely noise` | 候选更像噪声，类型为 `uncertain_peak_or_noise` 且置信度低于伪峰阈值。 |
| 红色叉号 | `artifact / interference` | 更像电尖峰、采集干扰或非色谱伪峰，不作为已确认色谱峰。 |
| 紫色空心方框（叠加在状态符号外） | `gentle broad peak candidate` | 高概率平缓宽峰候选；内部圆点或三角仍表示其确认/待复核状态。 |
| 青绿色空心向右/向左三角 | `confirmed boundary: start > / < end` | 仅用于确认峰；向右三角是算法起点，向左三角是算法终点，二者都朝向峰区间内部。 |

`artifact` 的中文含义是“干扰/伪峰”，不是一种真实峰类型，也不是图片文件的意思。它表示算法确实在该位置发现了显著信号事件，但综合形态证据认为该事件更可能来自电信号、瞬时脉冲、采集中断或其他非色谱原因。疑似噪声在 CSV 中仍属于 `artifact` 状态，但在图中使用独立的灰色菱形，便于和红色电干扰叉号区分。`confirmed_peaks.csv` 不包含这些事件。

确认峰文字的第一行是“类型/模板 + 峰置信度”，第二行是 `起 … / 止 … min`；其他需要说明的事件仍显示类型和置信度，但不显示起止时间。峰非常密集时，确认峰及其边界文字优先并始终保留，其次是模板峰、平缓峰、负峰和电干扰，普通待复核文字优先级最低。低优先级文字被省略不等于候选被删除，应以标记点和 `all_detected_features.csv` 为准。

## 统一置信度

`peak_confidence` 表示“该事件是色谱峰”的综合证据，范围 0～1；CSV 同时提供 `peak_confidence_percent`。默认确认阈值是 0.75，正峰和负峰共用同一阈值。

正峰基础权重：

| 分项 | 权重 |
|---|---:|
| 局部 SNR | 25% |
| 相对突出度 | 25% |
| 与参考峰宽的相似度 | 20% |
| 对称性 | 15% |
| 峰顶非平坦度 | 15% |

负峰基础权重：

| 分项 | 权重 |
|---|---:|
| 局部 SNR | 18% |
| 相对突出度 | 15% |
| 峰宽相似度 | 12% |
| 对称性 | 10% |
| 相对基线下探深度 | 20% |
| 原始信号双侧谷深 | 25% |

各项先转换到 0～1，再加权。高噪声、单点脉冲、平顶电干扰、超宽背景和峰间浅谷会施加显式惩罚。所有分项得分均写入结果 CSV，避免只给一个无法解释的最终数字。

`template_confidence` 是另一项独立指标，表示峰与 T1～T6 中某一保留时间模板的匹配程度。它不参与“峰是否存在”的最终确认，避免明显真峰仅因保留时间或低位边界偏移而被错误降级。

阈值位于 [detector_config.json](configs/detector_config.json)：

```json
{
  "confirmation_threshold": 0.75,
  "artifact_threshold": 0.45
}
```

在获得人工真值前，不建议仅凭当前样例继续降低确认阈值。

## 输出文件

默认输出目录为 `outputs/reliable_detector_results/`：

- `all_detected_features.csv`：所有事件及完整分项评分。
- `confirmed_peaks.csv`：综合置信度达到阈值的峰，包括合格负峰。
- `review_required.csv`：证据不足或形态有歧义的事件。
- `interference_candidates.csv`：所有 `status=artifact` 的事件，包括电尖峰、电干扰和低置信度疑似噪声。
- `detected_peaks_for_validation.csv`：确认与待复核事件，不包含 `artifact`，适合人工逐峰检查。
- `confidence_statistics.csv`：按峰类型和状态统计置信度数量、均值、中位数、最小值和最大值。
- `file_quality_summary.csv`：逐文件的峰数、噪声、漂移和置信度摘要。
- `validation_labels_template.csv`：可填写人工真值的完整表。
- `priority_validation_set.csv`：建议优先复核的事件。
- `learned_peak_template.json`：从五条基准谱学习的 T1～T6 模板。
- `plots/`：每条独立曲线的三层对照图，包括未处理原始谱线、基线诊断以及带置信度、分类符号和确认峰起止端点的检测信号。
