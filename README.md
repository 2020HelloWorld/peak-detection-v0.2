# ChromPeak 色谱峰检测工程 v0.5.2

这是一个针对一维色谱时间序列的 Python 验证工程。当前目标是先可靠完成预处理、峰位置检测和峰形类型判定，再利用人工真值迭代参数；暂不处理 C++/ARM 移植。

## 当前能力

- 动态噪声：局部一阶差分 MAD、局部 SNR、高噪声工况保护。
- 基线漂移：arPLS、双极性稳健基线及 rolling-ball 局部背景。
- 鼓包工况诊断：同时输出首尾漂移、全程起伏、非线性弯曲和持续时间，不再只凭起点与终点的差值判断。
- 背景残差保护：弱候选使用随机噪声、结构化波纹和基线模型不确定度共同形成有效噪声；疑似结构化背景残差保留用于审计，但不能自行越过确认线。
- 正峰与负峰：独立检测分支，达到统一置信度阈值的负峰可直接确认。
- 宽峰与鼓包：区分普通正峰、宽峰、鼓包背景上的峰及宽背景候选。
- 平缓峰：新增保守的平缓宽峰候选规则，单独标记但仍保留人工复核属性。
- 重叠峰：根据峰窗口交叠和模板宽度输出重叠/未分离候选。
- 电信号干扰：不再把 `impulse_overlap` 当作一票否决；联合峰宽、采样点数、对称性、升降段连续性与斜率、顶部能量集中度、面积形状、局部基线和去尖后的峰体保留率，分别输出峰体分与电干扰分。
- 结果解释：逐峰输出分项得分、综合峰置信度和模板匹配置信度；密集区域的文字标签会自动二维避让；确认峰在图中另标峰顶时间、基线校正后的有符号峰顶值，以及算法起止点和时间。

需求文档中的 5.1～5.6 六类问题、验收指标和数据盘点见 [需求梳理](docs/需求梳理_v1.md)。

## 完整算法流程

当前程序的实际处理主链如下。它是“信号处理 + 规则分类 + 参考模板”的组合算法，不是端到端神经网络：

```text
ZIP/CSV 原始数据
  → 格式校验与 SHA-256 精确去重
  → 从基准谱学习 T1～T6 保留时间/峰宽模板
  → 每条曲线进行孤立脉冲识别、三路基线估计和动态噪声估计
  → 由诊断基线计算首尾漂移、全程起伏、弯曲程度和鼓包持续时间
  → 构造正峰检测分支与保留正负号的检测分支
  → 正、负候选峰分别搜索
  → 计算峰顶、边界、宽度、突出度、SNR、面积、升降段和去尖反事实特征
  → 对正候选联合计算 peak_body_score 与 electrical_interference_score
  → 匹配参考模板并计算两类置信度
  → 判定普通峰、负峰、宽峰、重叠峰、鼓包、电干扰或噪声候选
  → 按 confirmed/review/artifact 分级
  → 输出 CSV、统计表和三层对照图
```

### 阶段 1：原始数据读取、校验与去重

1. 从 ZIP 中读取所有 `.CSV`，要求列名严格为 `x,Curvel`。`x` 是时间轴，`Curvel` 是原始检测器响应。
2. 将两列转换为浮点数，检查是否存在 `NaN/Inf`、采样点是否少于 5 个、时间轴是否严格递增。任一检查失败会直接报错，不会静默修补原文件。
3. 对 CSV 原始字节计算 SHA-256。内容完全相同的文件只处理一次，同时在 `manifest.json` 中保留重复文件分组。
4. `Curve.y` 始终保存原始 `Curvel` 数值。结果图第一层直接绘制 `Curve.x/Curve.y`，不会被后续预处理覆盖。

相关技术：Pandas CSV 解析、NumPy 数值校验、SHA-256 内容哈希。

### 阶段 2：从基准谱学习 T1～T6 模板

程序把目录名以 `1-` 开头的 5 条基准谱作为参考数据。基准谱先经过与普通数据相同的预处理和正峰候选检测，再保留满足以下条件的稳定候选：

- 局部 `SNR ≥ 8`；
- 相对突出度 `≥ 0.003`；
- 95% 峰宽位于 `0.025～0.80 min`。

随后只按峰顶保留时间进行 complete-linkage 层次聚类，聚类距离阈值为 `0.120 min`。一个聚类至少要得到 4 条不同基准谱支持，才会成为稳定模板。对每个模板统计：

- 保留时间中位数；
- 自适应保留时间容差，限制在 `0.045～0.22 min`；
- 峰宽中位数、允许的峰宽下限和上限；
- 对称性中位数和允许下限；
- 支持该模板的基准谱数量。

模板按保留时间排序后命名为 T1、T2……。T1～T6 只代表保留时间模板槽，不自动代表 H2、CO、CH4 等化学名称；化学名称必须通过 `configs/component_map.json` 配置。

相关技术：参考样本筛选、SciPy complete-linkage 层次聚类、中位数与范围稳健统计。

### 阶段 3：单条谱图预处理

#### 3.1 采样间隔和有效量程

采样间隔 `dt` 取相邻时间差的中位数。用于归一化的稳健信号量程定义为原始信号 `0.1%` 与 `99.9%` 分位数之差，避免单个极端值完全决定量程。

#### 3.2 仅识别孤立单点脉冲

对原始信号使用 3 点中值滤波得到局部参考，计算原始值与局部中值之差。脉冲阈值取下面两者的较大值：

- 残差稳健标准差的 8 倍；
- 整条曲线稳健量程的 `0.4%`。

超过阈值的孤立点只在正峰预处理分支中用 3 点中值替换。连续较宽的事件不会在此阶段删除，而是保留到后面的峰/干扰分类。原始数据数组和原始谱线图不做替换。

相关技术：3 点中值滤波、MAD 稳健标准差、脉冲掩码。

#### 3.3 三种基线/背景估计

程序同时计算三类背景信息：

1. **双极性稳健基线**：约 `1.0 min` 的中值滤波，再用约 `0.35 min` 的二阶 Savitzky–Golay 趋势平滑。它不优先偏向正峰或负峰，是负峰分支使用的基线。
2. **arPLS 基线**：`pybaselines.arpls`，当前 `λ=1e7`、最多 50 次迭代，用于跟随普通慢漂移。
3. **rolling-ball 局部背景**：约 `0.50 min` 的半窗口和约 `0.025 min` 的平滑半窗口，用于描述鼓包和局部背景变化。

强负峰可能把下包络基线长时间向下拉，因此正峰主基线取 `max(arPLS, 双极性基线)`。rolling-ball 保持为一条独立背景模型，不再和双极性基线取最大值；这样主基线与局部背景之间的差异才能作为可解释的基线模型不确定度证据，而不会被预先抹平。

相关技术：中值趋势基线、Savitzky–Golay、arPLS 非对称重加权最小二乘、rolling-ball 背景估计。

#### 3.4 构造检测分支

- **正峰分支**：`孤立脉冲清理后的信号 - 正峰主基线`，再进行约 `0.015 min` 窗口、三阶多项式的 Savitzky–Golay 轻度平滑。
- **鼓包辅助分支**：`孤立脉冲清理后的信号 - rolling-ball 局部背景`，用于局部背景证据。
- **带符号分支**：`原始信号 - 双极性基线`，再轻度平滑。该分支保留正负方向，负峰搜索使用其相反数。

这些分支是供检测使用的基线校正信号，不是高斯、洛伦兹或 EMG 峰函数拟合结果。

#### 3.5 鼓包工况诊断：四个互补指标

鼓包不能只看“终点减起点”。程序先构造一条仅用于工况诊断的基线：逐点取正峰主基线、独立 rolling-ball 背景和双极性基线的中位数，再用约 `0.20 min` 的二阶 Savitzky–Golay 窗口平滑。该诊断基线不会替代峰检测主基线。

设稳健信号量程为

```text
R = Q99.9%(raw) - Q0.1%(raw)
```

首端和末端水平分别取诊断基线前后窗口的中位数；窗口长度为全程的 2%，并限制在 `0.10～0.25 min`。四个文件级指标为：

| 字段 | 当前定义与含义 |
|---|---|
| `endpoint_drift_ratio` | `(末端中位水平 - 首端中位水平) / R`，保留正负号。它擅长发现单向漂移，但无法单独发现“先升后降”或“先降后升”的鼓包。 |
| `baseline_excursion_ratio` | `(max(诊断基线) - min(诊断基线)) / R`，描述采集全程基线走过的总幅度，不要求首尾不同。 |
| `baseline_curvature` | 先用首末端水平连成直线，再计算“诊断基线减该直线”的 `1%～99%` 分位差并除以 `R`。这是对非线性弯曲的稳健代理量，不是数学上的二阶导数，因而对采样率和局部微小噪声不那么敏感。 |
| `bump_duration` | 诊断基线上通过局部可靠性筛选的正向宽背景区间的最长持续时间，单位 min；没有可靠区间时为 0。 |

局部黄色宽背景区采用灵敏、强证据两级搜索。灵敏分支要求诊断基线突出度不小于 `0.005R`（全量程的 0.5%）、半高宽不小于 `0.50 min`、候选距离不小于 `0.30 min`；强证据分支的突出度门槛为 `0.03R`。区间边界采用 `rel_height=0.90`，相隔不超过 `0.20 min` 的区间合并。这样低幅但持续很宽的鼓包不会再仅因未达到 3% 全量程而漏标。

降低局部门槛后必须防止把窄而高的真实分析峰拖出的宽基线形状误当成鼓包。对突出度低于 `0.03R` 的低幅候选，程序比较原始信号与诊断基线的绝对差；满足以下任一条件即拒绝该宽背景候选：

- 候选峰顶处的信号泄漏大于 `0.03R`；
- 候选区间内，信号泄漏大于 `0.03R` 的采样点比例超过 10%。

通过筛选且持续至少 `0.50 min` 的局部区间生成 `bump_mask`，并令 `bump_flag=true`；`bump_count` 是合并后的区间数。若已找到可靠的闭合局部区间，程序不会再启用旧的开放区间兜底，避免 A14 一类样本在前段产生与真正鼓包无关的误标。只有没有任何可靠局部区间时，才允许评估开放区间兜底，并用 `bump_is_open` 记录。

文件级另输出 `bump_strength`：

- `none`：没有可靠正向宽背景区间；
- `candidate`：灵敏分支检出可靠宽区间，但尚未达到强鼓包证据；
- `strong`：存在可靠宽区间，并且全程起伏与弯曲同时达到 `baseline_excursion_ratio ≥ 0.05`、`baseline_curvature ≥ 0.04`，或至少一个保留区间的局部突出度达到 `0.03R`。

四个全局指标仍完整输出，用于判断整条曲线的背景工况和解释非平稳程度，但不再作为是否绘制某个局部黄色区间的唯一门槛。这套规则刻意把“局部哪里存在宽背景”“整条曲线的鼓包有多强”和“阴影中的某个候选是否是真峰”分成三个问题。

当前 `bump_duration/bump_mask` 专门描述色谱中通常所说的正向“鼓包”。负向宽凹陷仍会反映在 `baseline_excursion_ratio`、`baseline_curvature` 和非平稳背景判断中，但不会被这个正向鼓包区间字段强行命名为鼓包。

以 `A3-鼓包干扰.CSV` 为例，当前实现得到的 `endpoint_drift_ratio` 约为 `-0.040`，其绝对值没有达到旧的 5% 首尾漂移报警线；但 `baseline_excursion_ratio` 约为 `0.268`、`baseline_curvature` 约为 `0.277`、最长 `bump_duration` 约为 `6.52 min`，因此仍会被识别为明显鼓包工况。这正是新增全程起伏和弯曲指标的目的。数值会随基线参数和代码版本小幅变化，应以本次生成的 `file_quality_summary.csv` 为准。

本轮局部宽背景修正重点针对 A1、A2、A8、A10、A12 的低幅长持续区，以及 A14 约 3～5 min 的中段鼓包。其共同漏检原因不是首尾差值，而是旧局部门槛 `0.03R` 容易被同一文件中的高峰抬高。新灵敏分支覆盖这些宽区段，同时用分析峰泄漏比例防止把前段窄高峰本身扩成黄色区域。v0.5.1 正式运行得到的黄色区间如下；这些是算法边界，不是人工真值：

| 样本 | 黄色宽背景区间（min） | `bump_strength` |
|---|---:|---|
| A1 | 5.8700～9.7717 | `strong` |
| A2 | 2.9983～4.4683 | `candidate` |
| A8 | 4.0667～9.4317 | `strong` |
| A10 | 8.0600～9.6950 | `candidate` |
| A12 | 3.0033～8.6450 | `candidate` |
| A14 | 3.0967～5.2850 | `strong` |

A14 找到中段可靠闭合区间后不再运行旧开放兜底，因此此前约 0.08～0.71 min 的错误黄色区已经消失。最终仍应以每次重新运行产生的 `bump_mask`、图像和 CSV 为准。

#### 3.6 随时间变化的噪声与有效噪声估计

每个采样点的局部噪声取以下两种估计的较大值：

1. 短窗平滑残差在约 `0.40 min` 窗口内的局部 MAD；
2. 信号一阶差分在约 `0.60 min` 窗口内的局部 MAD。

因此基础随机噪声随时间变化，而不是整条曲线共用一个固定标准差。文件级噪声指标为基础随机噪声中位数除以稳健信号量程。若该指标同时高于基准谱中位噪声的 10 倍和 `5×10⁻⁵`，则标记为高噪声工况。

鼓包扣除后的平滑波纹并不一定表现为高频随机噪声，只用短窗 MAD 会把其噪声估得过低并产生虚高 SNR。为此，正向弱候选还计算：

- `structured_noise`：正峰检测信号减去约 `0.20 min` 中值趋势后的局部稳健波动尺度；
- `baseline_model_uncertainty`：正峰主基线与独立 rolling-ball 背景之差的局部稳健波动尺度；
- `baseline_model_disagreement`：两条背景模型逐点绝对差，仅作为解释性特征输出。

当候选的 `relative_prominence < 0.005` 时，实际评分使用的

```text
effective_noise = max(random_noise,
                      structured_noise,
                      baseline_model_uncertainty)
snr = prominence / effective_noise
```

较强候选仍用 `random_noise`，避免真实大峰把自身计入结构化噪声后反向压低自己的 SNR。CSV 同时保留 `random_snr` 和基于 `effective_noise` 的 `snr`，便于核查是哪一种噪声证据起了作用。

相关技术：滑动 MAD、稳健一阶差分噪声、局部 SNR、参考噪声归一化。

### 阶段 4：正峰和负峰候选检测

正峰使用蓝色正峰分支；负峰使用带符号分支的相反数。二者分别调用 SciPy `find_peaks`，当前最低候选条件为：

| 条件 | 当前规则 |
|---|---|
| 峰高 | `max(2 × 局部噪声, 0.0005 × 稳健信号量程)` |
| 突出度 | `max(3 × 局部噪声, 0.0005 × 稳健信号量程)` |
| 最小宽度 | 3 个采样点 |
| 最小峰间距离 | `max(3个采样点, 0.008 min)` |
| 突出度搜索窗口 | 最长约 `2.0 min` |

这意味着算法扫描完整时间范围，但不会把每一个局部极值都输出为候选。低于局部噪声门槛的微小起伏会在这一阶段被忽略。

相关技术：SciPy `find_peaks`、动态高度/突出度数组、正负双分支搜索。

### 阶段 5：逐候选特征计算

对每个候选计算下列特征：

| 特征 | 计算含义 |
|---|---|
| `apex_time_min` | 候选峰顶对应的原始时间坐标。 |
| `corrected_apex_value` | 峰顶处基线校正后的有符号值；正峰为正、负峰为负，也是确认峰标签中显示的“校正值”。 |
| `raw_apex_value` | 同一峰顶时间处由未处理原始谱线插值得到的响应值，便于和校正值对照。 |
| `start/end_time_min` | SciPy 突出度基准上相对高度 95% 处的近基线边界。它是算法边界，不是已人工确认的积分边界；检测图仅为 `confirmed` 峰显示这两个端点。 |
| `width_min` | 上述 95% 相对高度宽度。 |
| `fwhm_min` | FWHM（Full Width at Half Maximum，半高全宽）：在峰高 50% 处，峰左、右交点之间的时间差，单位为 min。它描述峰体宽窄；数值越小通常表示峰越尖，但不能脱离对称性、连续性、斜率和去尖后峰体保留等证据单独判定为毛刺。 |
| `width_samples/fwhm_samples` | 95% 宽度和半高宽对应的采样点数。它们保留用于审计采样充分性和比较不同采集率，当前不进入电形态评分，也不是峰顶尖点救回的硬门。 |
| `top_width_ratio` | 10% 相对高度处宽度与半高宽的比值，用于识别平顶/异常峰顶。 |
| `rise_time_min/fall_time_min` | 峰顶左、右半高段的上升和下降时间。 |
| `rise_continuity/fall_continuity` | 去尖后升、降段保持正确方向的步进比例，越接近 1 表示峰体越连续。 |
| `normalized_max_slope/max_step_ratio/slope_asymmetry` | 最大斜率相对峰高的归一化值、最大单步跳变占峰高的比例，以及升降斜率对称度。 |
| `top3_energy_fraction` | 峰窗口内绝对能量集中在最高三个采样点的比例。该字段易受采样率影响，当前只作审计，不进入电形态评分或硬门。 |
| `apex_core_area_fraction` | 以峰顶为中心固定物理时间 `±0.005 min` 内的正面积占整个自动峰面积的比例。它替代“最高三个采样点”进入评分，使不同采样率下的含义保持一致。 |
| `area_shape_factor` | 面积除以“突出度 × 半高宽”，用于检查峰体是否具有足够肩部和尾部面积。 |
| `height/prominence` | 在相应预处理检测分支上的峰高和突出度。 |
| `relative_prominence` | 突出度除以本文件稳健信号量程。 |
| `random_snr/snr` | 前者仅用局部随机噪声；后者对弱正候选使用 `effective_noise`，是进入置信度评分的 SNR。 |
| `structured_noise/effective_noise` | 鼓包波纹尺度，以及随机噪声、结构化噪声和基线模型不确定度的保守组合。 |
| `apex_in_bump/bump_overlap_fraction` | 峰顶是否落在诊断鼓包区间内，以及自动峰窗口与该区间的重合比例。 |
| `candidate_density/weak_candidate_density` | 峰顶前后 `0.50 min` 内全部/弱正候选的数量，包含候选自身。 |
| `area` | 在 95% 边界内，对基线校正检测信号的正值做梯形积分；当前不是最终定量面积。 |
| `symmetry` | 峰顶左右半高宽较小值除以较大值，越接近 1 越对称。 |
| `bilateral_depth` | 对轻度平滑后的原始谱线检查峰顶相对左右肩部是否同时突出/下探；负峰必须形成真实双侧谷形。 |
| `signed_depth_relative` | 带符号分支相对本文件量程的上升或下探深度。 |
| `baseline_change_ratio` | 候选范围内 rolling-ball 局部背景变化量相对峰高的比例。 |
| `impulse_overlap` | 峰顶 `±0.005 min` 内是否与暂定单点脉冲掩码重合；只是触发二次检查，不再单独否决候选。 |
| `apex_quadratic_excess_ratio` | 排除暂定异常点后，在物理时间窗口内对周围峰体做二次拟合；实际峰顶超出拟合预测的正向余量再除以突出度。达到 `0.015` 才构成可解释尖点证据。 |
| `apex_spike_flag` | 仅当 `impulse_overlap=true` 且 `apex_quadratic_excess_ratio >= 0.015` 时为真；普通窄峰相对三点中值偏高不会自动被当成尖点。 |
| `apex_impulse_ratio` | 三点中值暂定修复幅度相对突出度的比例，保留用于审计；当前联合评分和峰体硬门使用更稳健的 `apex_quadratic_excess_ratio`。 |
| `despiked_height_retention/despiked_prominence_retention/despiked_area_retention` | 临时修复尖点后，峰高、突出度和面积相对原事件的保留率。 |
| `peak_body_survives_despike` | 是否同时通过保守峰体门，即去掉尖点后仍存在完整、连续的峰体。 |
| `file_impulse_density_per_min` | 整条曲线每分钟的脉冲密度，用于识别随机毛刺工况。 |
| `electrical_assessment_applicable` | 当前联合电形态模型只适用于正候选。正候选为真；负候选为假。 |
| `peak_body_score` | 正候选的多参数峰体证据综合分，范围 0～1，越高越像真实色谱峰。负候选不适用，在 CSV 中为空值。 |
| `electrical_interference_score` | 正候选的多参数电干扰证据综合分，范围 0～1，越高越像电尖峰/毛刺。负候选不适用，在 CSV 中为空值。 |
| `edge_distance_min` | 峰顶距采集起点或终点的最小距离。 |

### 阶段 6：模板匹配与两种置信度

候选只在峰顶落入某个模板的保留时间容差时匹配该模板。模板匹配置信度 `template_confidence` 的组成是：保留时间 50%、峰宽 30%、对称性 15%、峰顶非平坦度 5%。它回答“该峰像不像 T1～T6 中的某个模板”。

峰存在置信度 `peak_confidence` 回答“该事件像不像真实色谱峰”。它与模板匹配置信度分开，避免明显峰因为保留时间漂移或暂时没有组分映射而被直接否定。峰宽得分会和所有参考模板宽度比较，取最相似者。

各证据先映射为 0～1 分数。SNR 在约 3～30 之间进行对数映射，相对突出度在正常门槛到 `0.03` 之间进行对数映射；峰宽按与参考宽度的对数比例评分，对称性、非平顶度和负峰下探深度分别归一化。正负峰的详细权重见后面的“统一置信度”。

当前会施加的主要惩罚包括：

- 高噪声且相对突出度不足：峰置信度乘 `0.75`；
- 靠近采集边缘：乘 `0.65`；
- 模板窗口内但峰顶平坦或不对称的异常窄峰：乘 `0.45`；
- 多参数联合判为明显电尖峰：乘 `0.25`，即使保留时间与模板重合也不会被模板“救回”；
- 可靠峰顶异常但既不能救回、也不足以判纯电尖峰：乘 `0.55` 并保留复核；
- 通过去尖反事实检验的“真实峰带峰顶尖点”：乘 `0.90`，保留峰但反映峰顶异常的不确定性；
- 超宽鼓包/基线候选：乘 `0.80`；
- 负峰与脉冲重合：乘 `0.65`；
- 两侧都有正峰的浅负向谷底：乘 `0.65`。
- 满足结构化背景残差联合门控：参考峰宽得分减半，并将峰置信度限制在确认阈值以下；该事件保留为待复核而不是删除。

### 阶段 7：峰类型分类

#### 7.1 正峰

落入模板保留时间窗口的候选：

- 峰宽和对称性均在模板范围内：`normal_positive_peak`；局部背景变化比例达到 0.25，或峰顶/至少一半自动峰窗口落入 `bump_mask` 时，改为 `positive_peak_on_hump`。
- 先运行独立于保留时间的电形态联合判别；强电干扰证据为 `electrical_interference_candidate`，去尖后峰体仍可靠则为 `peak_with_apex_spike`。其余比模板窄且峰顶偏平或明显不对称的候选仍为 `electrical_interference_candidate`，否则为 `narrow_positive_peak`。
- 比模板明显宽：`broad_or_overlapped_peak`。
- 其余形态不确定：`uncertain_peak_or_noise`。

不在任何模板窗口的候选：

- 多参数联合判为强电干扰，或检测到峰顶尖点但不能证明去尖后峰体仍可靠：`electrical_spike`。
- 宽度大于 `0.80 min` 时先检查平缓宽峰规则；满足规则则为 `gentle_broad_peak_candidate`，否则为 `broad_hump_or_baseline`。普通宽背景若最终置信度达到确认线，再改为 `broad_positive_peak`。
- 峰宽仍像某个参考峰且具有最低对称性：`unassigned_positive_peak`，表示确认有峰但保留时间未映射。
- 其余：`uncertain_peak_or_noise`。

模板窗口外极小的波纹会根据普通/高噪声相对突出度门槛直接过滤，避免输出大量假峰。模板窗口内的小峰仍保留竞争资格，防止弱组分被统一删除。

`gentle_broad_peak_candidate` 的中文标注是“平缓宽峰候选”。为了符合“很大概率”而不是把所有宽事件改名，当前采用高精度、低召回的保守条件：不匹配模板、无单点脉冲重合、95% 峰宽不小于 `0.80 min`、半高宽不小于 `0.30 min`、95% 峰宽与半高宽之比不大于 6、对称性不小于 0.25、峰顶宽度比不大于 0.65、相对突出度不小于 0.003、SNR 不小于 8、局部背景变化比例不大于 3、原始谱线双侧回落深度比例不小于 0.002，并且远离采集边缘。

该类型仍叫“候选”，不能直接解释为化学组分峰。真正的慢波动可能已经被基线算法吸收；要完整区分平缓峰和缓慢基线，还需要在扣基线前增加专门的低频背景检测分支和人工真值验证。

#### 7.2 负峰

负峰首先必须同时满足带符号下探深度和原始谱线双侧谷深门槛；高噪声时两个门槛都会提高。通过后：

- 宽度小于 `0.018 min`：`negative_electrical_spike`；
- 宽度不超过 `0.80 min`：`negative_peak`；
- 更宽：`broad_negative_peak`。

如果负向事件较浅，并且前后 `0.8 min` 范围内都有正峰，则改为 `interpeak_valley_or_negative_peak` 并降低置信度，避免把峰间谷底强行判成化学负峰。

#### 7.3 重叠和同模板多候选

- 同一个模板窗口内可以保留多个确认峰。模板匹配最好的候选保留原类型，其余标记为 `secondary_or_overlapping_candidate`，但不会仅因同槽多峰而撤销其峰存在状态。
- 相邻确认正峰的 95% 峰窗口发生交叠时，两者标记为 `overlapping_positive_peak`。

当前方法只做重叠关系识别，还没有进行多峰参数化解卷积。

#### 7.4 “正向毛刺/电尖峰”和“电信号干扰候选”

v0.5.2 不再使用“`impulse_overlap=true` 就判电尖峰”的单条件规则。三点中值差只生成暂定脉冲掩码；候选峰顶 `±0.005 min` 内与该掩码重合时，程序排除暂定异常点，在以异常点为中心、半宽 `max(0.010 min, 0.25×FWHM)` 的窗口中对周围峰体做二次拟合。只有无法由拟合峰体解释的正向超额占突出度达到 `0.015`，才令 `apex_spike_flag=true`。采样过稀且不足以拟合时还要求孤立斜率反转，避免把正常窄峰的尖顶误当成单点电毛刺。

评分使用固定物理时间的 `apex_core_area_fraction`：

```text
apex_core_area_fraction =
    峰顶前后各 0.005 min 内的正面积 / 自动峰区间总正面积
```

它不会因某台仪器每分钟采样点更多而天然变大。`fwhm_samples` 和 `top3_energy_fraction` 仍写入 CSV，专门用于检查采样充分性和审计顶部能量，但当前不参与两项形态评分，也不作为救回峰体的硬门。

两组分数仅对正候选计算：

- `peak_body_score`：FWHM/模板相对宽度支持 14%、升降斜率平衡 10%、对称性 12%、面积形状 16%、非平顶性 8%、局部基线稳定性 8%、升降连续性 12%、固定时间峰顶面积分散度 8%、去尖后峰高/突出度/面积平均保留率 8%、二次拟合峰顶超额较小 4%。
- `electrical_interference_score`：物理时间 FWHM 过窄 18%、固定时间峰顶面积过度集中 14%、归一化斜率跳变 14%、二次拟合峰顶超额 10%、文件级脉冲密度 18%、低对称性 6%、平顶异常 6%、升降斜率失衡 8%、去尖后峰体损失 6%。

正候选的“去尖后峰体仍存在”必须满足以下联合门。绝对宽度和模板相对宽度是二选一入口，其余条件全部必需：

```text
入口 A：fwhm_min >= 0.060 min 且 area_shape_factor >= 1.15
或
入口 B：存在模板，width_min 位于
        [0.75 × 模板宽度下限, 1.25 × 模板宽度上限]
        且 area_shape_factor >= 0.85

并且：
symmetry >= 0.75
top_width_ratio <= 0.45
baseline_change_ratio <= 0.25
min(rise_continuity, fall_continuity) >= 0.80
apex_core_area_fraction <= 0.35
despiked_height_retention >= 0.65
despiked_prominence_retention >= 0.65
despiked_area_retention >= 0.65
apex_quadratic_excess_ratio <= 0.25
```

最终按四种情况路由：

- `peak_with_apex_spike`：存在可靠尖点证据、峰体通过联合门、`peak_body_score >= 0.72`，且峰体分不低于电干扰分；峰置信度乘 `0.90`。
- 纯电尖峰：`electrical_interference_score >= 0.45`、峰体未通过联合门，并且存在可靠尖点或 `fwhm_min < 0.018 min`。模板外为 `electrical_spike`，模板内为 `electrical_interference_candidate`，峰置信度乘 `0.25`。
- 峰顶异常但证据仍有歧义：有可靠尖点，却既不满足上述救回条件也不满足纯电尖峰条件；保留为 `electrical_interference_candidate`，峰置信度乘 `0.55`，通常进入人工复核，而不是直接删除。
- 没有可靠峰顶异常：继续按模板、峰宽和背景关系分类为普通正峰、未映射正峰、鼓包上的正峰等。

模板只提供相对宽度支持，保留时间本身不能救回强电瞬态。相邻确认峰若窗口重叠，`window_overlaps_next_confirmed_peak/window_overlaps_previous_confirmed_peak` 仍写入 `reasons`，但不会覆盖 `peak_with_apex_spike` 这一质量类别。

当前联合电形态模型只用于正候选。负候选继续使用独立负峰/谷底/负向电尖峰规则；其 `electrical_assessment_applicable=false`，`peak_body_score`、`electrical_interference_score` 以及三项去尖保留率在 CSV 中留空，不能把空值解释为 0 分。

这些名称只说明波形形态，不能单凭谱线断言具体硬件故障一定来自地线或某个电子元件。

#### 7.5 结构化背景残差联合门控

`structured_background_residual`（结构化背景残差）表示：程序找到了一个局部正向起伏，但多项证据共同表明它更像鼓包、单向漂移或结构化噪声扣除后留下的连续波纹，而不是足以自动确认的独立色谱峰。它不是靠“峰高低于某一个阈值”单独决定，也不等同于严格的 `bump_flag`。

当前必须先满足文件级非平稳背景条件之一：

```text
baseline_excursion_ratio >= 0.08
或 baseline_curvature >= 0.04
或 abs(endpoint_drift_ratio) >= 0.05
```

并且同一正候选还必须同时满足全部局部条件：

```text
relative_prominence < 0.005
width_min < 0.80 min
前后 ±0.50 min 内的弱候选数 >= 2（包含自身）
bilateral_depth_relative < 0.0015
```

先完成电尖峰、电干扰和其他基础形态判定；已经判为 `artifact` 或电事件的候选不会被残差类别覆盖。其余通过联合门控的候选会重新计算减半模板宽度得分后的完整综合分，并把置信度上限设为 `confirmation_threshold - 0.01`；默认确认线为 0.75，因此上限为 0.74。其类型改为 `structured_background_residual`、状态保持 `review`，仍完整写入审计 CSV。该规则阻止弱而密集的背景波纹借助虚高 SNR 和偶然相似的模板宽度自动确认为峰，但不会删除数据，也不会替代人工判断。相邻残差峰顶间隔不超过 `0.35 min` 时归为同一 `R1/R2…` 波纹组，便于按区域复核。

### 阶段 8：状态分级

峰类型描述“它像什么”，状态描述“当前证据有多可靠”，两者不能混为一谈：

| 状态 | 当前规则 |
|---|---|
| `confirmed` | `peak_confidence ≥ 0.75`，正峰和负峰使用同一确认阈值。 |
| `artifact` | `peak_confidence < 0.45`，且类型属于电信号事件或 `uncertain_peak_or_noise`。 |
| `review` | 其他未达到确认线的事件，包括证据不足、宽峰/谷底歧义或低置信度负峰。 |

因此，`unassigned_positive_peak` 的“未映射”不等于未确认；它只表示尚未匹配 T1～T6/化学组分。`artifact` 表示检测到了信号事件但更像干扰或噪声，不表示程序没有在该位置检测到变化。`structured_background_residual` 则是保留在 `review` 中的专门审计类别，不能和已删除候选或已确认峰混淆。

### 阶段 9：结果输出与可视化

每条曲线都会生成三层图：未处理原始谱线、基线诊断、预处理检测信号及分类标记。诊断出的宽背景区间在第二、三层以淡黄色阴影显示；这表示“算法检测到正向宽背景候选”，不是把整段区域判成一个化学峰。第三层的浅绿色竖向矩形是从基准谱学到的 T1～T6 保留时间容差窗口，只是帮助判断候选是否落入某个模板槽；即使窗口内没有检测峰，它仍会显示，因此不能把绿色矩形理解成峰、峰边界或确认状态。结构化背景残差用小灰点显示，并减少逐点文字，避免长波纹区的标签再次互相遮挡。

第三层对每个 `confirmed` 峰在算法起点画空心向右三角、在算法终点画空心向左三角，并在标签中显示 `顶 x.xxxx min / 校正值 y` 和 `起 x.xxxx / 止 x.xxxx min`。其中“校正值”来自该峰相应检测分支的基线校正值，并按峰方向保留符号：正峰为正值、负峰为负值；它不等同于第一层原始谱线的绝对响应，原始峰顶响应另存于 `raw_apex_value`。`review`、`artifact`、`likely noise` 和结构化背景残差不画起止端点，也不附加这组确认峰专属信息。标签排版会读取 Matplotlib 实际文字边界框，在像素坐标中尝试上下和左右多个位置，避开其他标签、坐标轴边界和图例，并用细引导线连接峰顶。确认峰标签具有最高优先级并始终保留；密集区域如果确实没有无碰撞位置，只省略低优先级待复核文字。随后按状态拆分 CSV，并生成逐文件噪声/鼓包汇总、分类型置信度统计、模板 JSON 和人工真值填写模板。

需要特别注意：当前 `area` 和 `start/end_time_min` 是自动候选阶段的基线校正面积和算法边界，适合对比及人工复核，但还未经过人工积分边界和标准物质标定，不能直接当作最终浓度定量结果。`rel_height=0.95` 表示从峰顶向突出度基准下降 95% 的近基线交点，不是“峰高的 95%”；背景漂移、长尾或重叠时，自动区间可能偏宽，图中显示并不等于边界已经人工确认。

### 当前尚未完成的能力边界

- 未配置 `component_map.json` 前，T1～T6 不能自动解释为具体化学组分。
- 尚未进行高斯/EMG 等参数化峰拟合和重叠峰解卷积。
- 尚未用人工逐峰真值校准置信度，因此它是可解释的综合评分，不是严格统计概率。
- 尚未获得独立盲测集上的检出率、误报率、类型混淆矩阵和边界误差。
- 当前面积不是经过标准曲线、响应因子和人工积分边界校准的定量面积。
- 平缓宽峰目前是保守候选规则，尚未建立扣基线前的多尺度低频检测和真值闭环。
- 四个鼓包指标、局部宽背景筛选和结构化残差门控是针对现有样例的稳健工程规则，尚未在独立、逐峰标注的盲测集上完成阈值校准；`bump_flag` 说明存在可靠宽背景区间，`bump_strength` 说明证据等级，二者都不等于其中每个小起伏一定是假峰。

## 如何理解检测结果图

`plots/` 对每条去重后的独立曲线生成一张三层总览图。三层共享同一条时间轴，目的是让原始测量、基线处理和最终判定可以在相同位置逐层对照；内容相同的重复 CSV 不重复绘图，其对应关系保存在 `manifest.json`。

### 三层曲线

1. **第一层：完整原始谱线**
   - 深灰色 `raw samples (unprocessed)` 直接连接 CSV 的 `x,Curvel` 原始采样值。
   - 不平滑、不插值、不替换毛刺、不扣基线、不改变幅度，也不叠加检测标记。
   - 使用全曲线纵轴，是检查极端正/负事件以及验证下面处理层是否偏离原始数据的权威视图。
2. **第二层：基线诊断**
   - 浅灰色 `raw reference`：同一条原始谱线，仅作为位置参照。
   - 红色 `robust main baseline`：正峰检测使用的稳健主基线。
   - 紫色 `rolling-ball local background`：独立局部背景模型，用于鼓包和局部背景变化诊断。
   - 淡黄色阴影 `broad-background region (candidate/strong)`：诊断出的正向宽背景区间；它是背景工况，不是一个化学峰。
3. **第三层：检测与分类**
   - 蓝色 `preprocessed positive`：孤立脉冲暂定修复、主基线扣除和轻度 Savitzky–Golay 平滑后的正峰检测分支。
   - 灰色 `signed branch`：保留正负号的检测分支，主要供负峰搜索和确认。
   - 黑灰色水平零线：基线校正后的 0 参考。
   - 淡黄色阴影和第二层完全对齐，便于判断候选是否位于宽背景中。
   - 浅绿色竖向矩形是从五条基准谱学习的 T1～T6 保留时间容差窗口，不是检测峰、峰宽、积分边界或确认结果。

第三层不是高斯、洛伦兹或 EMG 参数拟合曲线，而是供候选检测和形态分析使用的预处理信号。峰顶和边界时间仍位于原始时间坐标。算法扫描完整时间范围并分别搜索正、负事件，但图中的标记只代表通过最低候选门槛的事件，不等同于原始曲线的每个局部极值；全部保留候选以 `all_detected_features.csv` 为准。

### 颜色、阴影和符号

| 符号 | 图例 | 含义 |
|---|---|---|
| 绿色圆点 | `confirmed` | 综合峰置信度达到确认阈值，当前默认阈值为 0.75。 |
| 橙色三角 | `review` | 有峰证据，但置信度不足或形态存在歧义，需要人工复核。 |
| 灰色菱形 | `likely noise` | 候选更像噪声，类型为 `uncertain_peak_or_noise` 且置信度低于伪峰阈值。 |
| 红色叉号 | `artifact / interference` | 更像电尖峰、采集干扰或非色谱伪峰，不作为已确认色谱峰。 |
| 灰色小点 | `structured background residual` | 非平稳背景上弱、密集且缺少清晰双侧峰形的结构化背景残差；保留为待复核，不进入确认峰。 |
| 紫色空心方框（叠加在状态符号外） | `gentle broad peak candidate` | 高概率平缓宽峰候选；内部圆点或三角仍表示其确认/待复核状态。 |
| 紫色空心加号形 `P`（叠加在状态符号外） | `chromatographic peak with apex spike` | 去掉峰顶尖点后仍保留可靠色谱峰体；当前类型为 `peak_with_apex_spike`。内部绿色圆点仍表示确认状态。 |
| 青绿色空心向右/向左三角 | `confirmed boundary: start > / < end` | 仅用于确认峰；向右三角是算法起点，向左三角是算法终点，二者都朝向峰区间内部。 |
| 浅绿色竖向矩形 | `T1–T6 template RT windows (not peaks)` | 基准谱学习出的 T1～T6 保留时间容差窗口；不是检测峰、峰宽、峰边界或确认结果。 |
| 淡黄色阴影 | `broad-background region (candidate/strong)` | 诊断基线上的正向宽背景候选区；`candidate` 为灵敏分支检出，`strong` 为强证据工况。阴影内仍可存在真实峰。 |

状态颜色和形态叠加符号回答不同问题：绿色/橙色/红色/灰色表示 `confirmed/review/artifact/likely noise` 的可靠性状态；紫色方框或紫色 `P` 表示平缓宽峰、带峰顶尖点峰等形态类别。因此一个峰可以同时看到绿色圆点和紫色 `P`，二者并不冲突。

`artifact` 的中文含义是“干扰/伪峰”。它表示算法确实发现了显著事件，但当前形态证据更支持电信号、瞬态脉冲、采集中断或其他非色谱来源；不表示程序没有检测到变化。低置信度 `uncertain_peak_or_noise` 在 CSV 中同属 `artifact`，但图上使用灰色菱形与红色电干扰叉号分开显示。`confirmed_peaks.csv` 不包含这些事件。

淡黄色阴影、浅绿色矩形和灰色小点含义不同：黄色阴影是诊断基线上的宽背景区间，绿色矩形是固定的模板保留时间容差窗口，灰点是逐候选层面的残差判定。黄色阴影中仍然允许存在证据足够强的真实峰；未被涂成黄色的候选也可能因整条曲线的非平稳背景和局部联合证据而进入残差审计类别。

### 峰顶文字、确认边界和密集标签

- 确认峰文字第一行是“类型/模板 + 峰置信度”，第二行是 `顶 … min / 校正值 …`，第三行是 `起 … / 止 … min`。
- “校正值”来自对应正/负检测分支：正峰为正，负峰为负；它不是第一层原始检测器响应，原始峰顶值另存于 `raw_apex_value`。
- 起点和终点来自突出度基准的自动近基线边界，仅对 `confirmed` 峰绘制青绿色空心三角；不是经过人工确认的最终积分边界。
- `review` 和 `artifact` 不显示确认峰专属边界。峰非常密集时，程序按实际文字边界自动上下/左右避让并用细引导线连接峰顶；确认峰文字必须保留，低优先级待复核文字可能隐藏，但候选标记和 CSV 行不会删除。

### `raw_zoom_plots/`：只缩放显示轴的原始分段图

当前绘图代码还为每条独立曲线生成 `raw_zoom_plots/` 分段原始图，并在 `raw_zoom_index.csv` 中记录路径、四段时间范围、样本数、显示纵轴和超出显示范围的原始点数。该图专门解决“全程存在很高峰时，后段小波动在总览原始图中看不清”的问题：

- 时间轴按物理时间等分为连续 4 段，每段单独绘制；
- 横纵数据仍是 CSV 原始 `x,Curvel` 采样值，不平滑、不插值、不去毛刺、不扣基线、不替换任何点；
- 每段只独立设置一个稳健的显示 `y` 轴，使用分位数和 IQR 抑制极端点对视野的压缩，再留 8% 边距；
- 超出该段显示纵轴的原始点仍存在于数据和绘图输入中，没有被裁剪或删除，标题会报告数量；
- 若要核查这些极端点，应返回 `plots/` 第一层的全程原始谱线。分段图只是观察工具，不能作为预处理结果或算法输入。

如果旧输出目录中尚无 `raw_zoom_plots/`，按当前 v0.5.2 代码重新运行后会生成该目录和 `raw_zoom_index.csv`。

## 统一置信度

`peak_confidence` 表示“该事件是色谱峰”的综合证据，范围 0～1；CSV 同时提供 `peak_confidence_percent`。默认确认阈值是 0.75，正峰和负峰共用同一阈值。

正峰基础权重：

| 分项 | 权重 |
|---|---:|
| 局部/有效 SNR | 25% |
| 相对突出度 | 25% |
| 与参考峰宽的相似度 | 20% |
| 对称性 | 15% |
| 峰顶非平坦度 | 15% |

负峰基础权重：

| 分项 | 权重 |
|---|---:|
| 局部 SNR | 18% |
| 相对突出度 | 15% |
| 峰宽相似度 | 12% |
| 对称性 | 10% |
| 相对基线下探深度 | 20% |
| 原始信号双侧谷深 | 25% |

各项先转换到 0～1，再加权。高噪声、联合判定的电干扰、超宽背景和峰间浅谷会施加显式惩罚；单独的 `impulse_overlap` 不再直接决定类别或置信度。弱正候选的 SNR 分项使用前述 `effective_noise`；结构化背景残差还会减半其模板宽度奖励并设置确认上限。`peak_body_score` 和 `electrical_interference_score` 是正候选的并行形态评分，不取代 `peak_confidence`。所有原始证据和分项得分均写入结果 CSV，避免只给一个无法解释的最终数字。

`template_confidence` 是另一项独立指标，表示峰与 T1～T6 中某一保留时间模板的匹配程度。它不参与“峰是否存在”的最终确认，避免明显真峰仅因保留时间或低位边界偏移而被错误降级。

阈值位于 [detector_config.json](configs/detector_config.json)：

```json
{
  "confirmation_threshold": 0.75,
  "artifact_threshold": 0.45,
  "electrical_spike_score_threshold": 0.45,
  "apex_spike_peak_body_floor": 0.72,
  "apex_spike_confidence_multiplier": 0.9,
  "hump_relative_prominence_floor": 0.005,
  "hump_weak_density_limit": 2,
  "hump_density_radius_min": 0.5,
  "hump_bilateral_depth_floor": 0.0015,
  "hump_candidate_width_limit_min": 0.8
}
```

在获得人工真值前，不建议仅凭当前样例继续降低确认阈值。

## 输出文件

默认输出目录为 `outputs/reliable_detector_results/`：

- `all_detected_features.csv`：所有事件及完整分项评分。
- `confirmed_peaks.csv`：综合置信度达到阈值的峰，包括合格负峰。
- `review_required.csv`：证据不足或形态有歧义的事件，包括保留下来的结构化背景残差。
- `interference_candidates.csv`：所有 `status=artifact` 的事件，包括电尖峰、电干扰和低置信度疑似噪声。
- `apex_spike_peaks.csv`：所有 `peak_with_apex_spike` 事件，集中给出峰体分、电干扰分、二次拟合峰顶超额和去尖保留率。
- `detected_peaks_for_validation.csv`：确认与待复核事件，不包含 `artifact`，适合人工逐峰检查。
- `confidence_statistics.csv`：按峰类型和状态统计置信度数量、均值、中位数、最小值和最大值。
- `structured_background_residuals.csv`：通过结构化背景残差联合门控的逐候选审计表，包含有效噪声、密度、背景重合和置信度等完整字段。
- `background_ripple_regions.csv`：把峰顶间隔不超过 `0.35 min` 的相邻残差合并为 `R1/R2…` 区域，输出起止时间、候选数、区域内最高置信度和最大相对突出度。
- `file_quality_summary.csv`：逐文件的峰数、噪声、置信度以及 `endpoint_drift_ratio`、`baseline_excursion_ratio`、`baseline_curvature`、`bump_duration`、`bump_flag`、`bump_strength`、`bump_is_open`、`bump_count`、`nonstationary_baseline_flag` 和稳健量程摘要。旧字段 `baseline_drift_fraction_of_range` 保留原定义：主基线精确末端与起点之差的绝对值除以原始全量程；新的 `endpoint_drift_ratio` 使用诊断基线端窗中位数和稳健量程，两者不要混用。判断 A3 这类首尾回归但中间隆起的工况，应联合查看局部区间、`bump_strength` 和四个全局指标。
- `validation_labels_template.csv`：可填写人工真值的完整表。
- `priority_validation_set.csv`：建议优先复核的事件。
- `learned_peak_template.json`：从五条基准谱学习的 T1～T6 模板。
- `plots/`：每条独立曲线的三层对照图，包括未处理原始谱线、基线诊断以及带置信度、分类符号、确认峰峰顶值和起止端点的检测信号；淡黄色阴影显示宽背景区间，浅绿色矩形显示 T1～T6 模板容差窗口，灰色小点显示结构化背景残差。
- `raw_zoom_plots/`：每条独立曲线的四段原始采样放大图；只改变各段显示纵轴，不处理或替换数据。
- `raw_zoom_index.csv`：分段原始图索引，记录路径、每段时间、样本数、显示纵轴范围和视野外原始点数。

配套成果文档位于 `docs/`：

- `技术总结_v0.5.2.md` / `技术总结_v0.5.2.docx`：完整技术流程、公式、阈值、结果、限制和扩展研究建议。

- `scripts/generate_deliverables.py`：从正式 CSV、三层检测图和原始分段图重新生成上述 Word 与 PPT。
