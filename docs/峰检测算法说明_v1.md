# 色谱峰检测与类型判定算法 v0.5.2

## 1. 当前交付结论

本版本已经按“先预处理、再找峰、最后判类型”的顺序跑完所给数据。输入共 45 个 CSV，识别到 4 组内容完全相同的重复文件，实际处理 41 条独立曲线。算法从 5 条基准谱自动学习出 6 个稳定正峰模板；5 条基准谱的 6 个模板峰均被确认检出。

当前输出的是峰的形态类型和位置，不会在缺少映射依据时猜测 H2、CO、CH4、CO2 等化学组分。组分名称需要在 `component_map.json` 中补齐 T1～T6 的对应关系。

由于异常数据没有逐峰人工真值，不能把“本次运行结果”直接宣称为最终正确答案。当前算法对正峰和负峰统一计算 0～1 综合峰置信度，达到默认阈值 0.75 即确认；负峰不再无条件进入待复核。峰是否存在的 `peak_confidence` 与模板/组分匹配的 `template_confidence` 分开输出。本版包含保守的平缓宽峰候选、密集标签自动避让，并只为确认峰在图中标出峰顶时间、基线校正后的有符号峰顶值，以及自动起止端点和时间。v0.5.2 还把正向电尖峰识别改为多参数联合判别，分别输出 `peak_body_score` 和 `electrical_interference_score`；`impulse_overlap` 不再一票否决完整峰体。

针对鼓包谱误报，当前实现新增四项文件级诊断指标、弱候选的有效噪声和结构化背景残差联合门控。它不会仅凭首尾差值判断鼓包，也不会把鼓包区间里的全部事件直接删除；可疑残差保留为 `review`，供人工逐一核验。

## 2. 算法处理流程

1. 数据检查：读取 `x,Curvel`，按内容哈希去除重复曲线，保留原文件来源。
2. 孤立脉冲预处理：仅替换达到强异常阈值的单点脉冲；较宽事件不直接删除，留给后面的干扰分类。
3. 基线校正：arPLS 负责慢漂移，双极性中位数基线防止负峰把 arPLS 长时间向下拖拽，二者取稳健上包络作为正峰主基线；rolling-ball 保持为独立局部背景模型，用于鼓包证据和基线模型不确定度。
4. 鼓包诊断：由三条背景模型的逐点中位数构造诊断基线，计算首尾漂移、全程起伏、非线性弯曲和宽背景持续时间，并生成局部鼓包区间。
5. 降噪：短窗 Savitzky–Golay 平滑；由平滑残差和局部一阶差分 MAD 共同估计随时间变化的随机噪声；弱正候选再叠加结构化波纹和基线模型不确定度，形成有效噪声。
6. 候选检测：SciPy `find_peaks` 分别检测正峰和负峰，计算峰顶时间、原始/基线校正峰顶值、起止、95% 峰宽、半高宽、突出度、局部 SNR、面积、对称性和相对幅度；图中只为确认峰显示峰顶时间与校正值及自动起止端点。
7. 模板学习：从 5 条基准谱学习 6 个峰的保留时间、允许漂移、宽度和对称性范围。
8. 类型判定：对正候选使用物理时间 FWHM、固定 `±0.005 min` 峰顶面积、升降段连续性/斜率、二次拟合峰顶超额、面积形状、局部基线和去尖保留率形成峰体分与电干扰分；采样点宽度和最高三点能量只作审计。随后综合模板、噪声工况、背景变化和相邻峰窗口完成最终分类。负候选不使用这套联合电形态分数，继续走独立负峰规则。
9. 可靠性分级：`confirmed/确认`、`review/待复核`、`artifact/干扰或伪峰`。

## 3. 对 5.1～5.6 的对应

| 文档问题 | 当前算法处理 |
|---|---|
| 5.1 动态噪声 | 局部 MAD 噪声、局部 SNR、整条曲线高噪声保护；高噪声时提高相对突出度门槛。 |
| 5.2 基线漂移 | arPLS + 双极性稳健基线 + 独立 rolling-ball 背景；同时输出首尾漂移、全程起伏、非线性弯曲和鼓包持续时间。首尾接近但中间明显起伏的曲线仍能报警。 |
| 5.3 平缓宽峰与基线 | 模板外、半高宽不小于 0.30 min 且满足双侧回落、对称性、SNR、背景变化等保守条件时，输出“平缓宽峰候选”；其他宽事件仍输出“宽鼓包、宽峰或基线”。弱而密集的非平稳背景残差使用联合门控保留为待复核，不强行确认为组分峰。 |
| 5.4 重叠峰/大峰上的小峰 | 用 95% 峰窗口交叠、模板峰宽包络、rolling-ball 局部背景变化和 `bump_mask` 重合，输出“重叠正峰”“宽峰或未分离重叠峰”“鼓包或漂移背景上的正峰”。 |
| 5.5 电信号干扰 | 单点脉冲重合只触发二次拟合复核；使用物理时间 FWHM、固定 `±0.005 min` 峰顶面积、峰形对称性、升降连续性/斜率、面积形状、局部基线、文件脉冲密度、二次拟合峰顶超额和去尖保留率联合区分纯电尖峰、带峰顶尖点的色谱峰及普通峰。强电形态即使与模板保留时间重合也不会被模板救回。 |
| 5.6 负峰 | 独立双极性分支；要求原始曲线呈双侧谷形且相对基线确实下探。峰间浅谷会标为“峰间谷底或负峰”，避免强行确认。 |

## 4. 鼓包诊断与结构化残差规则

### 4.1 稳健量程与诊断基线

所有鼓包比例先用原始信号的稳健量程归一化：

```text
R = Q99.9%(raw) - Q0.1%(raw)
```

这样能降低单个极端采样点对归一化比例的支配。诊断基线按以下顺序构造：

1. 逐点取正峰主基线、独立 rolling-ball 背景和双极性基线的中位数；
2. 使用约 `0.20 min` 的二阶 Savitzky–Golay 窗口平滑。

诊断基线只用于判断文件的背景工况，不替换正峰或负峰检测分支的基线。

### 4.2 四个文件级指标

首端和末端水平不是取单个端点，而是取诊断基线前后窗口的中位数。窗口为全程的 2%，并限制在 `0.10～0.25 min`。

| 字段 | 定义 |
|---|---|
| `endpoint_drift_ratio` | `(末端水平 - 首端水平) / R`，保留方向。只能直接反映首尾净变化。 |
| `baseline_excursion_ratio` | `(max(诊断基线) - min(诊断基线)) / R`，反映全程最大起伏，即使首尾接近也不会归零。 |
| `baseline_curvature` | 用首末端水平连成直线；计算诊断基线相对该直线残差的 `Q99%-Q1%`，再除以 `R`。它是稳健的非线性弯曲代理量，不是二阶导数。 |
| `bump_duration` | 诊断基线上通过局部可靠性筛选的正向宽背景区间的最长时间跨度，单位 min；没有可靠区间时为 0。 |

局部宽背景检测分成两级：

- 灵敏分支：突出度不小于 `0.005R`（全量程的 0.5%）；
- 强证据分支：突出度不小于 `0.03R`；
- 两个分支都要求半高宽不小于 `0.50 min`、候选距离不小于 `0.30 min`，区间采用 `rel_height=0.90` 的左右边界；相隔不超过 `0.20 min` 的区间合并。

降低局部门槛后，窄而高的真实分析峰可能通过诊断基线产生宽形状泄漏。为避免把这种泄漏涂成鼓包，对突出度低于 `0.03R` 的低幅候选增加排除规则：若候选峰顶处的 `abs(原始信号 - 诊断基线)` 大于 `0.03R`，或候选区间内该绝对差大于 `0.03R` 的采样点比例超过 10%，则拒绝该宽背景候选。强证据分支仍保留，避免真正强鼓包被局部高峰影响而删除。

通过筛选且持续至少 `0.50 min` 的闭合局部区间生成 `bump_mask`，并令 `bump_flag=true`；`bump_count` 为合并后的区间数。只要已经存在可靠闭合局部区间，就不再运行旧开放区间兜底，以免生成与真正鼓包无关的远端阴影。只有没有可靠局部区间时才允许评估开放区间，并用 `bump_is_open` 记录。

`bump_strength` 对文件级证据分三级：

- `none`：没有可靠正向宽背景区间；
- `candidate`：灵敏分支找到可靠区间，但未达到强证据；
- `strong`：存在可靠区间，并且 `baseline_excursion_ratio ≥ 0.05` 与 `baseline_curvature ≥ 0.04` 同时成立，或至少一个保留区间的突出度达到 `0.03R`。

四个全局指标仍完整输出并用于解释整体背景工况，但不再是生成局部黄色区间的唯一门槛。`bump_flag/bump_mask` 描述是否及在哪里存在可靠正向宽背景，`bump_strength` 描述证据强弱；它们都不代表阴影区内每个局部起伏都是假峰。

当前 `bump_duration/bump_mask` 专门描述正向宽隆起。负向宽凹陷仍会提高 `baseline_excursion_ratio`、`baseline_curvature`，并可能触发非平稳背景判断，但不会被正向鼓包区间字段强行标成鼓包。

### 4.3 为什么 A3 首尾漂移小仍能检出鼓包

`A3-鼓包干扰.CSV` 的基线中段先明显上升、随后下降，首尾差值会发生抵消。当前实现得到的近似结果为：

```text
endpoint_drift_ratio      ≈ -0.040
baseline_excursion_ratio  ≈  0.268
baseline_curvature        ≈  0.277
bump_duration             ≈  6.52 min
```

所以旧式“首尾差达到 5% 才报警”会漏掉它，而全程起伏、非线性弯曲和持续时间会触发 `bump_flag`。这些数值是当前参数下的算法输出，代码或基线参数变化后可能小幅改变，应以实际生成的 `file_quality_summary.csv` 为准。

### 4.4 弱候选的有效噪声

短时间随机噪声由平滑残差和局部一阶差分 MAD 的较大值估计。但鼓包扣除后留下的平滑波纹可能让该随机噪声过小，从而产生虚高 SNR。正向候选因此另有：

- `structured_noise`：正峰检测信号减去约 `0.20 min` 中值趋势后的局部稳健波动尺度；
- `baseline_model_uncertainty`：正峰主基线与独立 rolling-ball 背景之差的局部稳健波动尺度；
- `baseline_model_disagreement`：两条背景模型的逐点绝对差，作为解释性字段。

对 `relative_prominence < 0.005` 的弱正候选：

```text
effective_noise = max(random_noise,
                      structured_noise,
                      baseline_model_uncertainty)
snr = prominence / effective_noise
```

较强候选仍使用 `random_noise`，避免真实大峰把自身抬高的结构化尺度反过来当作噪声。CSV 同时输出 `random_snr` 与最终 `snr`。

### 4.5 结构化背景残差联合门控

`structured_background_residual` 不等同于严格的鼓包标志，也不是由单一峰高阈值决定。它覆盖鼓包、单向漂移或结构化噪声等非平稳背景；文件必须先满足以下条件之一：

```text
baseline_excursion_ratio >= 0.08
或 baseline_curvature >= 0.04
或 abs(endpoint_drift_ratio) >= 0.05
```

同一正候选还必须同时满足：

```text
relative_prominence < 0.005
width_min < 0.80 min
前后 ±0.50 min 内弱候选数 >= 2（包含自身）
bilateral_depth_relative < 0.0015
```

电尖峰、电干扰和基础 `artifact` 形态优先，不会被残差标签覆盖。其余通过门控的候选重新计算减半参考宽度得分后的完整综合分，峰置信度最多为 `confirmation_threshold - 0.01`；默认确认阈值为 0.75，因此残差最高为 0.74，状态保持 `review`。它不会从结果中删除：逐候选记录写入 `structured_background_residuals.csv`；峰顶间隔不超过 `0.35 min` 的相邻残差归为 `R1/R2…` 区域并写入 `background_ripple_regions.csv`。

该联合门控的目的，是让弱、密集、双侧峰形不清且处在非平稳背景文件中的波纹不能仅凭随机噪声估计过小或峰宽偶然接近模板而自动确认。明显、孤立、具有清晰双侧峰形的事件不会只因为文件有鼓包就被统一降级。

### 4.6 图中的显示方式

- 第二层基线诊断图和第三层检测图用淡黄色阴影标出 `bump_mask` 宽背景区间，图例注明 `candidate/strong`；阴影是背景工况，不是化学组分峰。
- 第三层浅绿色竖向矩形是 T1～T6 模板的保留时间容差窗口。它们来自基准谱模板，会固定显示在相应时间范围，不代表该处已经检测到峰，也不是峰的起止边界或确认结果。
- 结构化背景残差用灰色小点 `structured background residual` 显示，并避免为每个密集小波纹都排一条文字标签。
- 绿色圆点仍表示达到确认线的峰；确认峰标签另显示 `顶 … min / 校正值 …` 以及 `起 … / 止 … min`。校正值来自该峰相应检测分支，并按峰方向保留符号：正峰为正、负峰为负；原始响应另存于 `raw_apex_value`。结构化残差保持待复核，不画确认峰起止三角。

## 5. 正向电信号干扰的多参数联合判别

### 5.1 为什么不能只看 `impulse_overlap`

单点脉冲掩码只能说明峰顶附近存在一个相对三点中值异常的采样点，不能说明整个事件都是电干扰。正常窄峰也可能高于三点中值；多采样点电瞬态反而未必触发单点掩码。因此 `impulse_overlap` 只启动第二阶段检查，不直接决定类别。

对峰顶 `±0.005 min` 内的暂定异常点，程序排除掩码点，在半宽 `max(0.010 min, 0.25×FWHM)` 的物理时间窗口内用周围点拟合二次曲线。定义：

```text
apex_quadratic_excess_ratio =
    max(0, 实际异常点值 - 二次拟合预测值) / prominence
```

只有 `impulse_overlap=true` 且该比例不小于 `0.015` 才令 `apex_spike_flag=true`。可用点不足时，程序采用相邻肩部预测，并额外要求孤立斜率反转，保护粗采样的真实窄峰。

每个正候选新增并联合使用以下证据：

- 物理时间支持：`fwhm_min` 和模板相对宽度；
- 峰形：`symmetry`、`top_width_ratio`、`area_shape_factor`；
- 升降段：`rise_time_min/fall_time_min`、`rise_continuity/fall_continuity`、`normalized_max_slope`、`max_step_ratio`、`slope_asymmetry`；
- 峰顶能量：以峰顶为中心固定 `±0.005 min` 的 `apex_core_area_fraction`；
- 尖点和工况：`apex_spike_flag`、`apex_quadratic_excess_ratio`、`file_impulse_density_per_min`；
- 局部背景：`baseline_change_ratio`；
- 去尖反事实：`despiked_height_retention`、`despiked_prominence_retention`、`despiked_area_retention`。

`width_samples/fwhm_samples`、`top3_energy_fraction` 和三点中值形成的 `apex_impulse_ratio` 继续输出用于审计采样充分性与旧证据，但不进入当前两项评分，也不是峰体救回硬门。

### 5.2 两个独立形态分数

`peak_body_score` 范围 0～1，越高越像有肩部、尾部和连续升降段的真实峰体。当前权重为：物理时间 FWHM/模板相对宽度支持 14%、升降斜率平衡 10%、对称性 12%、面积形状 16%、非平顶性 8%、局部基线稳定性 8%、升降连续性 12%、固定时间峰顶面积分散度 8%、去尖后峰高/突出度/面积平均保留率 8%、二次拟合峰顶超额较小 4%。

`electrical_interference_score` 范围 0～1，越高越像电尖峰/毛刺。当前权重为：物理时间 FWHM 过窄 18%、固定时间峰顶面积集中 14%、归一化斜率跳变 14%、二次拟合峰顶超额 10%、文件级脉冲密度 18%、低对称性 6%、平顶异常 6%、升降斜率失衡 8%、去尖后峰体损失 6%。这两个分数用于解释和形态路由，不等同于最终 `peak_confidence`。

固定时间峰顶面积定义为：

```text
apex_core_area_fraction =
    integral(max(signal, 0), apex-0.005, apex+0.005)
    / integral(max(signal, 0), automatic_peak_start, automatic_peak_end)
```

边界采用线性插值，因此指标由分钟而不是采样点个数定义，更适合比较不同仪器采集率。

### 5.3 去尖反事实和保守峰体门

程序用相同平滑、相同区间和相同面积规则对“含暂定尖点分支”与“暂定修复分支”做同口径比较。峰体入口有两条，满足其中之一即可；其余条件必须全部满足：

```text
绝对入口：fwhm_min >= 0.060 min 且 area_shape_factor >= 1.15
或
模板相对入口：存在模板，width_min 位于
  [0.75 × width_low_min, 1.25 × width_high_min]
  且 area_shape_factor >= 0.85

symmetry >= 0.75
top_width_ratio <= 0.45
baseline_change_ratio <= 0.25
min(rise_continuity, fall_continuity) >= 0.80
apex_core_area_fraction <= 0.35
despiked_height_retention >= 0.65
despiked_prominence_retention >= 0.65
despiked_area_retention >= 0.65
apex_quadratic_excess_ratio <= 0.25
```

通过上述门只表示“去尖后有可靠峰体”。要标成 `peak_with_apex_spike`，还必须有 `apex_spike_flag`、`peak_body_score >= 0.72`，并且峰体分不低于电干扰分。

### 5.4 四种分类路径

- `electrical_spike`：电干扰分不小于 0.45、峰体未通过联合门，并且存在可靠尖点或 `fwhm_min < 0.018 min`；模板窗口内同样的强电形态命名为 `electrical_interference_candidate`。峰置信度乘 `0.25`，保留时间不能将其救回。
- `peak_with_apex_spike`：有可靠尖点，去尖后峰体通过全部联合门，峰体分不小于 0.72 且不低于电干扰分；峰置信度乘 `0.90`。
- 歧义峰顶异常：有可靠尖点，但既不能救回为峰、也不足以判为纯电尖峰；命名为 `electrical_interference_candidate`，峰置信度乘 `0.55` 并保留复核。
- 普通色谱峰：没有需要单列的峰顶尖点，继续按模板、峰宽和背景关系进入普通正峰、未映射正峰或鼓包上的峰等原类别。

相邻确认峰窗口重叠时，原因仍写入 `reasons`，但不会覆盖 `peak_with_apex_spike`。当前联合电形态只适用于正候选；负候选的 `electrical_assessment_applicable=false`，三个去尖保留率、`peak_body_score` 和 `electrical_interference_score` 在 CSV 中为空，而不是 0，负候选继续使用独立负峰/谷底/负向电尖峰规则。

## 7. 输出文件怎么用

- `outputs/reliable_detector_results/detected_peaks_for_validation.csv`：人工逐峰检查的主表，包含确认和待复核事件。
- `outputs/reliable_detector_results/confirmed_peaks.csv`：综合峰置信度达到阈值的正峰和负峰。
- `outputs/reliable_detector_results/review_required.csv`：宽峰、重叠、鼓包、结构化背景残差及噪声歧义事件。
- `outputs/reliable_detector_results/interference_candidates.csv`：电尖峰和电信号干扰候选。
- `outputs/reliable_detector_results/apex_spike_peaks.csv`：集中列出 `peak_with_apex_spike` 及其峰体、电干扰和二次拟合指标。
- `outputs/reliable_detector_results/confidence_statistics.csv`：按类型和状态汇总综合置信度。
- `outputs/reliable_detector_results/priority_validation_set.csv`：建议优先人工核验的集合，已排除纯“峰或噪声”低优先级项。
- `outputs/reliable_detector_results/validation_labels_template.csv`：可直接填写人工真值的完整模板，末尾预留人工位置、类型、组分和备注列。
- `outputs/reliable_detector_results/all_detected_features.csv`：全部输出及数值特征。
- `outputs/reliable_detector_results/structured_background_residuals.csv`：结构化背景残差的逐候选审计表；候选没有被删除，仍可检查峰顶、有效噪声、局部密度、置信度和判定原因。
- `outputs/reliable_detector_results/background_ripple_regions.csv`：按 `R1/R2…` 合并相邻残差，输出区域起止、候选数、最高置信度和最大相对突出度。
- `outputs/reliable_detector_results/file_quality_summary.csv`：逐文件的噪声、峰数、异常数，以及四个鼓包指标、`bump_flag/bump_strength/bump_is_open/bump_count`、`nonstationary_baseline_flag` 和稳健量程。兼容字段 `baseline_drift_fraction_of_range` 仍沿用旧定义：主基线精确末端与起点之差的绝对值除以原始全量程；新的 `endpoint_drift_ratio` 则使用诊断基线端窗中位数和稳健量程，两者不要混用。A3 这类首尾回归的鼓包应联合查看局部区间、`bump_strength` 和四个全局指标。
- `outputs/reliable_detector_results/learned_peak_template.json`：从基准谱学习的 T1～T6 模板。
- `outputs/reliable_detector_results/plots/`：每条独立曲线的未处理原始信号、基线诊断和检测标记三层图；淡黄色阴影表示宽背景区间，浅绿色矩形表示 T1～T6 模板保留时间容差窗口，灰色小点表示结构化背景残差；确认峰以空心向右/向左三角分别标出算法起点/终点，标签给出 `顶 … min / 校正值 …` 和 `起 … / 止 … min`；密集标签按实际文字宽度分配到上下轨道并用引导线连接峰顶。
- `outputs/reliable_detector_results/raw_zoom_plots/`：把每条原始曲线按物理时间等分四段，各段独立设置显示纵轴；绘制值仍是未经任何处理的 `x,Curvel`。
- `outputs/reliable_detector_results/raw_zoom_index.csv`：原始分段图索引及每段时间、样本数、显示纵轴和视野外原始点计数；正式目录当前包含 41 条索引及对应 41 张分段图。

主表关键字段：

- `apex_time_min/start_time_min/end_time_min`：峰顶和自动边界位置，单位 min。边界来自突出度基准上 `rel_height=0.95` 的近基线交点；它尚未经过人工边界真值校准，不能直接视为最终积分边界。
- `corrected_apex_value`：峰顶处基线校正后的有符号值；正峰为正、负峰为负，也是确认峰图中文字显示的“校正值”。
- `raw_apex_value`：同一峰顶时间处从未处理原始谱线插值得到的响应值，用于和校正值对照。
- `feature_type_cn`：中文峰类型。
- `status_cn`：确认、待复核、干扰/伪峰。
- `template_slot/component`：模板槽和可选组分名称。
- `width_min/fwhm_min`：95% 宽度和半高宽，单位 min；电形态评分使用物理时间宽度。
- `width_samples/fwhm_samples`：对应采样点宽度，仅用于审计采样充分性，不参与当前电形态评分和峰体硬门。
- `prominence/random_snr/snr/symmetry/area`：突出度、噪声比、对称性和自动面积；`random_snr` 只用随机噪声，弱正候选的最终 `snr` 使用 `effective_noise`。
- `rise_time_min/fall_time_min`：左右半高段的上升与下降时间。
- `rise_continuity/fall_continuity`：去尖后升降段保持正确方向的步进比例。
- `normalized_max_slope/max_step_ratio/slope_asymmetry`：归一化最大斜率、最大单步跳变占峰高的比例和升降斜率对称度。
- `top_width_ratio/area_shape_factor`：峰顶宽度，以及面积相对“突出度×半高宽”的峰体形状因子。
- `top3_energy_fraction`：最高三个采样点能量占比，只用于审计，避免采样率改变评分含义。
- `apex_core_area_fraction`：峰顶固定 `±0.005 min` 内正面积占自动峰总正面积的比例，进入两项电形态评分和峰体门。
- `impulse_overlap/apex_spike_flag`：前者只触发二次拟合；后者要求二次拟合超额达到 `0.015`，不是三点中值差的一票否决。
- `apex_quadratic_excess_ratio`：实际异常点高于周围二次拟合峰体预测的正向超额除以突出度。
- `apex_impulse_ratio`：旧三点中值暂定修复幅度比，仅用于审计。
- `despiked_height_retention/despiked_prominence_retention/despiked_area_retention`：临时去尖后峰高、突出度和面积的保留率。
- `peak_body_survives_despike`：候选是否通过去尖后的保守峰体硬门。
- `file_impulse_density_per_min`：整条曲线每分钟的脉冲密度。
- `electrical_assessment_applicable`：正候选为真，负候选为假。
- `peak_body_score/electrical_interference_score`：正候选的 0～1 峰体证据分与电干扰证据分；二者不等同于最终峰置信度。负候选不适用，在 CSV 中为空。
- `relative_prominence`：相对本文件量程的突出度。
- `random_noise/structured_noise/baseline_model_uncertainty/effective_noise`：随机、结构化、基线模型和最终有效噪声证据。
- `apex_in_bump/bump_overlap_fraction`：峰顶是否位于鼓包区，以及候选窗口和鼓包区的重合比例。
- `candidate_density/weak_candidate_density`：峰顶前后 `0.50 min` 内全部/弱正候选数，包含候选自身。
- `structured_background_residual_flag/ripple_group`：是否通过联合门控，以及所属 `R1/R2…` 残差区域。
- `peak_confidence/peak_confidence_percent`：峰存在的综合置信度。
- `template_confidence/template_confidence_percent`：T1～T6模板匹配置信度。
- `score_*`：SNR、突出度、峰宽、对称性、非平顶、负峰下探和双侧谷深的0～1分项得分。
- `reasons`：完整判定原因和阈值。

当前回归套件包含 20 项：保护 H1/H2/H3/A4 已知电干扰，覆盖 A7 带尖点峰体、与 T1 重合的合成毛刺、正常窄 T1、叠加单点尖刺的 T1、多采样率高斯峰、五份参考 T1/T2、F1 正负峰、B10 高噪声、鼓包目标点、确认边界和原始分段图。原始分段图测试明确检查等时分段以及只改变显示纵轴、不改变输入原始值。

## 8. 运行方式

当前目录已包含输入数据的工作副本，直接运行：

```powershell
python run_detector.py
```

也可以指定原始压缩包和新输出目录：

```powershell
python run_detector.py --zip "D:\色谱峰处理算法\一些典型的谱图.zip" --out outputs\reliable_detector_results
```

缺少依赖时：

```powershell
python -m pip install -r requirements.txt
```
