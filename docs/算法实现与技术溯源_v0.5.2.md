# ChromPeak 算法实现与技术溯源说明

> 适用版本：`chrompeak-detector 0.5.2`  
> 对应生产入口：`src/chrompeak/detector.py::main()`  
> 编制日期：2026-07-19  
> 文档目的：说明当前正式检测算法用了什么技术、借鉴了哪些公开算法、项目做了哪些工程改造，以及每项实现位于哪一段代码。

## 1. 结论先行

当前正式算法不是一个从 GitHub 直接下载后原样运行的单体算法，也不是神经网络模型。它是一条可审计的“一维信号处理 + 参考模板 + 规则分类 + 综合评分”流水线：

```text
ZIP/CSV 原始数据
  → 格式校验、时间轴校验、SHA-256 去重
  → 单点脉冲仅作暂时修复并保留脉冲证据
  → arPLS 主基线 + 双极性稳健基线 + rolling-ball 局部背景
  → Savitzky–Golay 短窗平滑
  → 局部一阶差分 MAD + 平滑残差 MAD + 结构化背景噪声
  → 正峰分支和负峰分支分别动态检峰
  → 峰宽、FWHM、突出度、SNR、对称性、斜率、面积等特征
  → 从五条标准谱学习 T1～T6 保留时间/宽度/对称性模板
  → 峰存在置信度、模板匹配分、峰体分、电干扰分
  → 正峰、负峰、平缓宽峰、鼓包上小峰、电尖峰、背景残差等分类
  → confirmed / review / artifact 分级及 CSV、图片、人工真值模板
```

其中：

- **直接采用的开源算法/实现**：pybaselines 的 arPLS、rolling-ball；SciPy 的 Savitzky–Golay、`find_peaks`、`peak_widths`、中值滤波和层次聚类。
- **基于通用思想自行实现的部分**：双极性基线、局部一阶差分 MAD 噪声、多背景模型不确定度、鼓包四指标、正负峰双分支、峰体/电干扰联合评分、综合置信度及类型路由。
- **仅用于早期比较、未进入正式主链的算法**：hplc-py 的 SNIP/偏斜峰混合拟合、SciPy CWT 检峰、pybaselines SNIP。
- **当前仍未完成的能力**：重叠峰解卷积与面积分配、正负重叠定量、用充分人工真值校准置信度为统计概率、跨机型自动迁移。

## 2. 如何理解“借鉴算法”

本项目把技术来源分成三层，三者不能混为一谈。

| 层级 | 含义 | 当前项目实例 |
|---|---|---|
| 直接调用开源库 | 运行时通过公开 Python API 使用该库实现，没有把库源码复制进项目 | `Baseline.arpls`、`Baseline.rolling_ball`、`scipy.signal.find_peaks`、`peak_widths`、`savgol_filter`、`median_filter`、`fclusterdata` |
| 借鉴公开技术思想后自研 | 使用公开的统计或信号处理原理，阈值、分支、组合逻辑和输出字段由本项目实现 | MAD 噪声、正负双分支、动态门槛、峰形评分、双侧谷深、鼓包诊断、去尖反事实比较 |
| 仅试验比较 | 曾在 `experiments/` 中跑过，用于判断可行性，但不参与 `outputs/reliable_detector_results` 的正式结果 | hplc-py、CWT、SNIP、多算法一致性比较 |

因此，“借鉴 hplc-py”不等于正式算法调用 hplc-py；当前生产主链没有使用 hplc-py 拟合结果，也没有让多个开源算法投票决定最终峰。

## 5. 原始数据读取、校验和去重

### 5.1 输入约束

`read_curves()` 逐个读取 ZIP 中的 CSV，并要求：

- 列名严格为 `x,Curvel`；
- 时间和值均为有限数；
- 至少五个采样点；
- 时间轴严格递增；
- 原始 CSV 字节计算 SHA-256。

代码：`src/chrompeak/core.py` L42～L65。

### 5.2 去重

算法按 SHA-256 只保留一条独立曲线参与检测，同时在 `manifest.json` 记录重复组。这样同一数据换文件名不会重复计数或重复参与模板学习。

代码：`src/chrompeak/detector.py` L247～L255；主流程调用在 L2546～L2549。

## 6. 噪声与基线预处理

这一阶段是当前算法可靠性的核心。目标不是把曲线“修得好看”，而是构造多个用途不同且可交叉检查的信号分支。

### 6.1 稳健量程

每条曲线先计算：

```text
R = Q99.9%(y) - Q0.1%(y)
```

`R` 用于把峰突出度、谷深和基线变化归一化。它比 `max(y)-min(y)` 更不容易被单个极端采样点支配。

代码：`detector.py` L544～L548；鼓包诊断中复算于 L304～L306。

### 6.2 孤立单点脉冲：只作暂时修复，不直接判死刑

生产代码先用长度为 3 的中值滤波得到 `median3`，计算：

```text
r3[i] = y[i] - median3[i]
threshold = max(8 × robust_sigma(r3), 0.004 × R)
impulse_mask[i] = |r3[i]| > threshold
```

只有命中的单点在 `impulse_clean` 检测副本中替换为中值。原始 `curve.y` 从不改写，`impulse_mask` 也继续进入后面的电干扰形态判断。

这样设计是因为：中值异常只能说明“峰顶局部很尖”，不能说明整段峰体不存在。A7 和合成 T1 用例证明，真实色谱峰上可以叠加一个单点电尖点。

代码：

- 全局 MAD：`detector.py` L263～L266；
- 单点暂时修复：`detector.py` L549～L556；
- 后续二次拟合峰顶超额：`detector.py` L477～L541；
- 去尖前后峰体保留率：`detector.py` L758～L802。

### 6.3 三种基线/背景模型

#### 6.3.1 arPLS 主基线

arPLS 属于迭代重加权惩罚最小二乘。其基本形式可写为：

```text
min_z  Σ w_i (y_i - z_i)^2 + λ ||D²z||²
```

第一项让基线接近观测，第二项惩罚基线的二阶差分以保证平滑；权重在迭代中根据负残差分布更新，使正向峰对基线的拉动减弱。

本项目调用：

```python
arpls_baseline, _ = fitter.arpls(impulse_clean, lam=1e7, max_iter=50)
```

代码：`detector.py` L562～L570。

#### 6.3.2 双极性稳健基线

`symmetric_baseline()` 先做约 `1.0 min` 中值滤波，再做约 `0.35 min` 二阶 Savitzky–Golay 平滑。该分支对正负方向使用同一处理，不优先把负峰当基线。

它有两个作用：

1. 作为负峰检测的基线；
2. 防止强负峰把 arPLS 下拉很长时间。

代码：`core.py` L68～L76；调用在 `detector.py` L558～L560。

#### 6.3.3 rolling-ball 局部背景

pybaselines 的 rolling-ball 实质是形态学开运算后平滑，形成类似“球从曲线下方滚过”的局部背景。本项目设置：

```text
half_window ≈ 0.50 min
smooth_half_window ≈ 0.025 min
```

它不直接替代正峰主基线，而作为独立的局部背景证据：

- 判断峰是否位于鼓包/漂移背景上；
- 计算主基线与局部背景的差异；
- 计算基线模型不确定度。

代码：`detector.py` L570～L581。

#### 6.3.4 为什么主正峰基线取上包络

```text
positive_main_baseline = max(arPLS_baseline, signed_baseline)
```

强负峰会把下包络型基线向下拖，导致负峰后形成大段假正信号。取二者逐点较大值可以保护正峰分支，而负峰仍从独立 `signed_baseline` 分支检测。

代码：`detector.py` L575～L590。

### 6.4 Savitzky–Golay 平滑

正峰、rolling-ball 背景分支和双极性分支均使用约 `0.015 min` 的三阶 Savitzky–Golay 平滑。窗口由实际采样间隔换算并强制为合法奇数，因此不同采样率下尽量保持相同物理时间尺度。

与简单移动平均相比，Savitzky–Golay 在局部用低阶多项式最小二乘拟合，通常更有利于保留峰顶和斜率形状。它只用于检测分支；第一层原始图仍直接画 CSV 采样点。

代码：`detector.py` L582～L590；窗口工具 `core.py` L34～L39。

### 6.5 动态随机噪声：两种估计取较大值

#### 6.5.1 平滑残差 MAD

对 `raw - smooth` 在约 `0.40 min` 局部窗口中计算：

```text
sigma_residual[i] = 1.4826 × median(|r - local_median(r)|)
```

并用全局稳健尺度的一半及 `R × 10^-6` 作为下限，防止平滑参考谱出现不现实的近零噪声。

代码：`detector.py` L263～L276。

#### 6.5.2 局部一阶差分 MAD

`local_noise_sigma()` 对一阶差分 `d[i]=s[i]-s[i-1]` 在约 `0.60 min` 窗口内计算 MAD：

```text
sigma_diff[i] = MAD(d) / (0.67448975 × sqrt(2))
```

`0.67448975` 把正态分布的 MAD 换算为标准差；一阶差分的方差是独立同方差噪声的两倍，所以再除以 `sqrt(2)`。

代码：`core.py` L79～L92。

#### 6.5.3 最终随机噪声

每个检测分支逐点取：

```text
random_noise[i] = max(sigma_residual[i], sigma_diff[i])
```

这样既能响应高频噪声，又不会在过于平滑的曲线上把 SNR 无限放大。

代码：`detector.py` L592～L606。

### 6.6 结构化噪声和基线不确定度

鼓包扣除后的平滑波纹不一定表现为高频随机噪声。如果只用短尺度 MAD，这些波纹会得到虚高 SNR。项目为正峰分支另外计算：

- `structured_noise`：正峰校正信号减去约 `0.20 min` 中值趋势后的局部稳健波动；
- `baseline_model_disagreement`：主基线与 rolling-ball 的逐点绝对差；
- `baseline_model_uncertainty`：两条背景模型之差的局部 MAD。

对相对突出度低于 `0.005R` 的弱正候选：

```text
effective_noise = max(random_noise,
                      structured_noise,
                      baseline_model_uncertainty)
SNR = prominence / effective_noise
```

较强候选仍使用 `random_noise`，避免真实大峰把自身缓慢形状误算为噪声。

代码：

- 结构化噪声：`detector.py` L437～L443；
- 三类噪声和模型差：`detector.py` L607～L613；
- 首次有效噪声：`detector.py` L866～L885；
- 结合候选密度重新计算：`detector.py` L1353～L1383。

### 6.7 文件级高噪声工况

```text
global_noise = median(noise_positive) / R
high_noise = global_noise > max(10 × reference_noise, 5e-5)
```

高噪声标志不会整条曲线一票否决。它只提高弱候选的相对突出度要求、提高负峰双侧谷深要求，并对仍很弱的候选乘 `0.75`。因此 B10 的明显真实峰仍能被确认。

代码：`detector.py` L621～L624、L1297～L1300、L1348～L1350、L1502～L1516、L1681～L1696。

## 7. 鼓包、漂移与平缓背景预处理

### 7.1 诊断基线

鼓包诊断不直接使用某一条基线，而是逐点取：

```text
diagnostic_baseline = median(main_baseline,
                             rolling_baseline,
                             signed_baseline)
```

再用约 `0.20 min` 二阶 Savitzky–Golay 平滑。这样单一模型的偏差不容易直接决定工况。

代码：`detector.py` L290～L313。

### 7.2 四个文件级指标

| 指标 | 定义 | 解决的问题 |
|---|---|---|
| `endpoint_drift_ratio` | `(末端窗口中位数 - 首端窗口中位数) / R` | 单向线性漂移 |
| `baseline_excursion_ratio` | `(max(diagnostic)-min(diagnostic))/R` | 首尾接近但中间先升后降的鼓包 |
| `baseline_curvature` | 诊断基线相对首末连线残差的 `Q99-Q1`，再除以 `R` | 非线性背景变化；避免直接二阶导数的采样率敏感性 |
| `bump_duration` | 最长可靠正向宽背景区间持续时间 | 鼓包持续尺度 |

端点水平使用全时长 2% 的窗口中位数，并限制窗口在 `0.10～0.25 min`。A3 即使首尾差小，仍会因全程起伏和弯曲被识别。

代码：`detector.py` L314～L328。

### 7.3 局部宽背景区间

在诊断基线上寻找正向宽背景，门槛为：

```text
prominence >= 0.005 × R
FWHM >= 0.50 min
candidate distance >= 0.30 min
boundary rel_height = 0.90
```

低于 `0.03R` 的弱宽背景还要检查原始信号相对诊断基线的窄峰泄漏；若峰顶超额大于 `0.03R`，或区间内超过该值的点多于 10%，则不把它当背景。相邻间隔不超过 `0.20 min` 的区间合并。

这套规则用于 A1、A2、A3、A8、A10、A12、A14 等鼓包场景；淡黄色区域来自 `bump_mask`，表示宽背景候选，不表示整段都是化学峰或都是伪峰。

代码：`detector.py` L330～L434。

### 7.4 非平稳背景和结构化残差门控

文件满足任一条件即视为非平稳背景：

```text
baseline_excursion_ratio >= 0.08
or baseline_curvature >= 0.04
or abs(endpoint_drift_ratio) >= 0.05
```

局部候选只有同时满足以下条件，才被标成 `structured_background_residual`：

```text
文件为非平稳背景
relative_prominence < 0.005
width < 0.80 min
±0.50 min 内弱候选数 >= 2
bilateral_depth_relative < 0.0015
```

它不会被删除，而是：宽度得分贡献减半、峰存在置信度上限设为 `0.74`，因此保留为 `review`，供人工验证。明显电干扰分类优先于背景残差；真实鼓包上峰也不会因为位于黄色区间就被统一拒绝。

代码：`detector.py` L1386～L1415、L1498～L1501、L1594～L1622。

## 8. 正峰与负峰候选识别

### 8.1 两条独立检测分支

```text
positive_work = smooth(impulse_clean - positive_main_baseline)
negative_work = -(smooth(raw - signed_baseline))
```

正峰在正向校正信号上找极大值；负峰把有符号校正信号取负后再找极大值。负峰不是把正峰结果简单改符号，也不使用会被强负峰拖动的正峰主基线。

代码：`detector.py` L656～L684。

### 8.2 动态候选门槛

对每个时间点构造动态门槛：

```text
height[i]     >= max(2 × local_noise[i], 0.0005 × R)
prominence[i] >= max(3 × local_noise[i], 0.0005 × R)
width         >= 3 samples
distance      >= 0.008 min
prominence search window <= 2.0 min
```

SciPy 的 `find_peaks` 支持与信号等长的高度和突出度数组，因此噪声升高区的门槛会随时间提高，而不是整条曲线使用一个绝对阈值。

代码：`detector.py` L685～L700。

### 8.3 峰宽、FWHM 和自动起止

同一候选计算三种宽度：

| 字段 | SciPy 参数 | 本项目含义 |
|---|---|---|
| `width_min` | `rel_height=0.95` | 从峰顶沿突出度下降 95% 的近基线宽度；图上的自动起止来自这里 |
| `fwhm_min` | `rel_height=0.50` | 相对突出度的半高宽 FWHM |
| `top_width_ratio` | `rel_height=0.10` 宽度 / FWHM | 峰顶尖锐或平顶程度 |

注意：这里的 FWHM 是相对 SciPy 峰突出度基准计算，不一定等于“原始纵轴最大值的一半”。边界使用插值交点，再换算为分钟。

代码：`detector.py` L701～L720；确认峰边界规格在 L1927～L1960 生成，并在 L2191、L2243～L2272 绘制。

## 10. 标准谱模板 T1～T6

### 10.1 模板训练样本

只有目录名以 `1-` 开头的五条参考谱参与学习。正候选必须满足：

```text
SNR >= 8
relative_prominence >= 0.003
0.025 min <= width_min <= 0.80 min
```

代码：`detector.py` L983～L995。

### 10.2 聚类和稳健参数

候选只按峰顶时间进行完全连接层次聚类，聚类距离阈值 `0.120 min`。每簇至少由 `max(4, 参考谱数量-1)` 条参考谱支持。每个模板记录保留时间中位数、允许时间误差、宽度中位数/上下限、对称性中位数/下限。

模板按保留时间从早到晚命名为 T1、T2……。T1～T6 是**当前这组标准数据中的相对模板槽**，不是跨机型天然相同的化学名称。不同机型若保留时间、柱型、流量或检测器不同，应分别训练模板或建立对齐模型。

代码：`detector.py` L999～L1039；模板匹配 L1042～L1103。

### 10.3 峰存在与组分匹配分开

- `peak_confidence`：证据是否支持“这里存在一个色谱峰”。
- `template_confidence`：候选是否像某个 T 槽，权重为保留时间 50%、宽度 30%、对称性 15%、非平顶 5%。

一个峰可以存在置信度很高但没有模板槽；也可以恰好落入模板时间窗，却因电尖峰形态而被判为干扰。模板保留时间不能单独救回强电干扰。

代码：`detector.py` L1093～L1103、L1475～L1593。

## 11. 峰存在置信度

### 11.1 得分归一化

SNR 和相对突出度使用对数区间映射至 0～1：

```text
SNR:                  3 → 0，30 → 1
relative_prominence:  0.0008 → 0，0.03 → 1
```

宽度分使用候选宽度与所有模板宽度中位数的最优对数比例；对称性和非平顶分使用线性裁剪。实现函数为 `clip01()`、`log_evidence()`、`log_ratio_score()`、`best_width_score()`。

代码：`detector.py` L1066～L1092、L1259～L1277。

### 11.2 正峰基础权重

```text
peak_confidence_positive =
    0.25 × SNR_score
  + 0.25 × prominence_score
  + 0.20 × width_score
  + 0.15 × symmetry_score
  + 0.15 × nonflat_score
```

代码：`detector.py` L1279～L1286。

### 11.3 负峰基础权重

```text
peak_confidence_negative =
    0.18 × SNR_score
  + 0.15 × prominence_score
  + 0.12 × width_score
  + 0.10 × symmetry_score
  + 0.20 × signed_depth_score
  + 0.25 × bilateral_raw_valley_score
```

代码：`detector.py` L1287～L1295。

### 11.4 正负峰为什么使用不同证据和权重

正负峰使用同一个确认阈值 `0.75`，但观测风险不对称：

- 正峰主基线针对正向峰优化，参考谱也主要提供正峰的宽度/形状分布，因此 SNR、突出度和模板宽度证据更可靠；
- 负峰来自独立双极性基线。强正峰被基线扣除后可能在两侧产生负旁瓣，单看负向 SNR 和宽度容易把旁瓣误认为负峰；
- 因此负峰降低模板宽度、SNR、突出度权重，增加 `signed_depth` 和“原始谱在峰顶左右两侧都更高”的 `bilateral_raw_valley` 权重；
- 这不是“负峰更难确认”或无条件降级，而是要求负峰通过与其误报机理相对应的额外证据。达到 0.75 的负峰同样直接标为确认。

这组权重是工程化证据组合，尚未用大规模人工真值做概率校准。

### 11.5 乘法修正和确认分级

| 条件 | 修正 |
|---|---:|
| 高噪声且突出度仍低于高噪声门槛 | `×0.75` |
| 靠近采集边界 | `×0.65` |
| 强电尖峰 | `×0.25` |
| 尖点歧义、峰体不足 | `×0.55` |
| 峰体保留但峰顶有尖点 | `×0.90` |
| 宽鼓包/宽基线候选 | `×0.80` |
| 负候选与脉冲重合 | `×0.65` |
| 浅负谷且两侧有正峰 | `×0.65` |
| 结构化背景残差 | 宽度证据减半，最终上限 `0.74` |

状态规则：

```text
peak_confidence >= 0.75                  → confirmed
peak_confidence < 0.45 且为电干扰/噪声 → artifact
其他                                     → review
```

因此 `peak_confidence` 应称为“0～1 的概率式综合证据分”，不能在无真值校准时解释为“实际正确概率 83%”。

代码：`detector.py` L1297～L1324、L1592～L1622、L1697～L1755；配置在 `configs/detector_config.json`。

## 12. 毛刺和电信号干扰的联合判别

### 12.1 为什么不能只看峰宽

真实窄色谱峰、电干扰和“色谱峰上叠加单点尖点”都可能有很小的采样点宽度。因此当前算法把“候选峰体证据”和“电瞬态证据”分开计算。

### 12.2 峰体分 `peak_body_score`

| 分项 | 权重 | 主要含义 |
|---|---:|---|
| 物理时间宽度/模板宽度支持 | 14% | FWHM 是否具备持续峰体 |
| 升降斜率平衡 | 10% | 两侧速度是否协调 |
| 对称性 | 12% | 两侧半宽是否接近 |
| 面积形状因子 | 16% | 面积是否由完整峰体贡献 |
| 非平顶 | 8% | 排除明显方顶/平顶 |
| 局部基线稳定 | 8% | 背景变化是否过强 |
| 升降连续性 | 12% | 是否存在连续上升和下降 |
| 峰顶能量分散 | 8% | 能量是否分布在峰体而非单点 |
| 去尖保留率 | 8% | 去掉可疑尖点后峰是否仍在 |
| 二次拟合峰顶超额较小 | 4% | 峰顶能否被邻域平滑峰体解释 |

代码：`detector.py` L1121～L1166。

### 12.3 电干扰分 `electrical_interference_score`

| 分项 | 权重 | 主要含义 |
|---|---:|---|
| 时间上极窄 | 18% | FWHM 是否接近瞬态 |
| 峰顶能量集中 | 14% | 固定物理时间核心面积占比 |
| 斜率跳变 | 14% | 单位时间最大步进是否过大 |
| 二次拟合无法解释的尖点 | 10% | 可疑点超出邻域峰体多少 |
| 文件脉冲串密度 | 18% | 整条曲线是否频繁出现孤立脉冲 |
| 低对称性 | 6% | 峰形是否偏斜破碎 |
| 平顶 | 6% | 是否像采集饱和/方波干扰 |
| 升降斜率失衡 | 8% | 一侧陡、一侧异常 |
| 去尖损失 | 6% | 去尖后高度/突出度/面积是否消失 |

代码：`detector.py` L1168～L1203。

### 12.4 去尖后峰体存活联合门

候选必须先满足一个入口：

```text
绝对入口：FWHM >= 0.060 min 且 area_shape_factor >= 1.15
或
模板入口：宽度处于 0.75×模板下限～1.25×模板上限，
          且 area_shape_factor >= 0.85
```

并且全部满足：

```text
symmetry >= 0.75
top_width_ratio <= 0.45
baseline_change_ratio <= 0.25
rise/fall continuity >= 0.80
apex_core_area_fraction <= 0.35
despiked height/prominence/area retention 均 >= 0.65
apex_quadratic_excess_ratio <= 0.25
```

随后：

- 峰体存活、峰体分至少 0.72 且不低于电干扰分 → `peak_with_apex_spike`；
- 电干扰分至少 0.45、峰体不存活，且有尖点或 `FWHM<0.018 min` → 纯电尖峰；
- 其余尖点歧义 → 电干扰候选，进入复核或干扰分级。

代码：`detector.py` L1205～L1256；分类优先级 L1475～L1593。

### 12.5 采样率不变性

分类尽量使用分钟、固定 `±0.005 min` 峰顶积分和每分钟脉冲密度，而不是“几个点宽”。`fwhm_samples`、`top3_energy_fraction` 仍输出用于审计，但不作为关键分类门。

回归测试在 60、120、600 samples/min 的合成高斯峰上验证类型一致，代码：`tests/test_detector.py` L239～L297。

## 13. 峰类型路由

### 13.1 正峰

分类顺序先处理电形态，再处理模板和宽峰，避免保留时间恰好匹配的电干扰被模板“救回”。主要类型如下：

| 类型 | 核心规则 |
|---|---|
| `normal_positive_peak` | 落入模板窗口，宽度和对称性符合模板，且无强电形态 |
| `positive_peak_on_hump` | 正峰成立，同时局部基线变化大或与 `bump_mask` 重合至少一半 |
| `narrow_positive_peak` | 落入模板但比模板窄；仍有较好峰形，未达到电干扰条件 |
| `unassigned_positive_peak` | 不在模板槽，但宽度支持和对称性足够 |
| `peak_with_apex_spike` | 单点尖点存在，但去尖后完整峰体通过联合门 |
| `electrical_spike` / `electrical_interference_candidate` | 电形态强，或尖点存在但峰体不足 |
| `broad_or_overlapped_peak` | 模板窗口内显著宽于模板，可能是宽峰或未分离重叠峰 |
| `gentle_broad_peak_candidate` | 模板外宽度/FWHM/SNR/对称性/双侧回落等满足保守平缓峰规则 |
| `broad_hump_or_baseline` | 宽事件但不足以可靠区分宽峰与背景 |
| `structured_background_residual` | 非平稳背景上的弱、密集、浅双侧证据残差 |
| `uncertain_peak_or_noise` | 证据不足，不能可靠归类 |

正峰路由代码：`detector.py` L1461～L1679；平缓宽峰函数 L1437～L1458。

### 13.2 平缓宽峰保守规则

`gentle_broad_peak_candidate` 只在模板外使用，并要求：

```text
无脉冲重合
width >= 0.80 min
FWHM >= 0.30 min
width/FWHM <= 6
symmetry >= 0.25
top_width_ratio <= 0.65
relative_prominence >= 0.003
SNR >= 8
baseline_change_ratio <= 3
bilateral_depth_relative >= 0.002
远离采集边界
```

这仍是“候选”而非已完成定量的宽组分峰。缓慢真实组分可能被基线吸收；完整解决需要带真值的低频峰专用验证和拟合分支。

### 13.3 负峰

负候选先通过硬门：

```text
正常噪声：bilateral_depth_relative >= 0.005
          signed_depth_relative >= 0.008
高噪声：  bilateral_depth_relative >= 0.010
          signed_depth_relative >= 0.020
```

再计算负峰置信度：

- `width < 0.018 min` → 负向电尖峰，置信度 `×0.25`；
- `0.018～0.80 min` → 普通负峰；
- `>0.80 min` → 宽负峰；
- 与脉冲掩码重合 → `×0.65`；
- 深度低于 5% 量程且前后 0.8 min 内都有正峰 → `interpeak_valley_or_negative_peak`，再 `×0.65`。

负峰代码：`detector.py` L1681～L1755。

### 13.4 重叠关系

- 同一模板槽有多个确认候选时，保留峰存在结论，较低模板匹配者标为 `secondary_or_overlapping_candidate`；
- 相邻确认正峰的自动起止窗口相交时，标为 `overlapping_positive_peak`；
- 当前仅完成“候选关系标记”，尚未做多峰曲线拟合、面积拆分或重叠正负峰联合定量。

代码：`detector.py` L1757～L1799。

### 15.2 当前仍在代码中的参数

arPLS 的 `lam=1e7`、各物理时间窗口、评分分项权重和大部分形态门当前仍写在代码中。它们可以审计，但还没有全部外移到 JSON。后续若进行跨机型标定，建议把以下内容分组配置：

- 采样/平滑和基线时间尺度；
- 候选最小距离、宽度和突出度；
- 正负峰置信度权重；
- 峰体分、电干扰分权重；
- 各类型硬门和乘法修正；
- 每台机型或每个通道的模板策略。

### 17.2 这些测试能证明什么

它们能证明：当前实现对已知关键样本和合成边界条件没有回退；正负峰、鼓包、尖点峰体、高噪声和图形输出符合当前规则。

它们不能证明：660 个候选全部分类正确，也不能给出召回率、误报率、面积误差或置信度校准误差。原因是异常数据尚无逐峰人工真值。

### 17.3 可靠性验证建议

下一步应使用 `validation_labels_template.csv` 建立逐峰真值，至少标注：

- 是否真实色谱峰；
- 峰类型；
- 峰顶、起止时间；
- 组分名称；
- 毛刺/鼓包/重叠关系；
- 可接受的面积参考。

然后按文件分组交叉验证，避免同一或重复曲线同时出现在调参和验证集合。建议报告：峰级 precision、recall、F1，峰顶时间误差，起止边界误差，面积相对误差，正/负峰分类型混淆矩阵，以及置信度可靠性曲线。

## 18. 当前限制与后续算法方向

1. **置信度尚未概率校准**：权重来自技术需求、标准谱分布和代表样本回归，不是监督学习拟合出的后验概率。
2. **模板是数据集/机型相关的**：T1～T6 不能在不同机型间直接当作相同化学组分。
3. **重叠峰尚未解卷积**：当前能标记重叠或宽峰，不能可靠拆面积。
4. **平缓峰与慢基线仍有不可辨识区**：无真值时，基线算法吸收的低频组分峰无法仅靠图形证明。
5. **负峰电形态模型未与正峰完全统一**：v0.5.2 的峰体/电干扰双分只适用于正候选；负峰沿用有符号深度和双侧原始谷证据。
6. **部分阈值仍硬编码**：跨机型应用前需要配置化和重新标定。
7. **面积是候选级工程值**：尚未进行标准物质响应、积分边界真值和不确定度闭环。

如果取得足够人工真值，可保留当前特征工程和可解释分支，用逻辑回归、梯度提升树或单调约束模型重新学习权重并校准概率；这比直接替换成黑盒深度网络更容易在现有小样本条件下审计。
