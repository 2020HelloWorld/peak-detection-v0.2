# ChromPeak v0.5.2 技术总结

## 1. 文档定位

本文面向后续算法扩展、人工验证、科研汇报和工程移植，说明 ChromPeak v0.5.2 从原始色谱采样到预处理、候选检测、峰形分类、置信度分级和结果可视化的完整技术链。本文以正式输出目录 `outputs/reliable_detector_results/` 为统计依据。

需要先明确：当前结果是可解释的工程判定，不是经过大规模逐峰真值校准后的统计概率。算法已经能够稳定输出候选位置、方向、形态类别、自动边界和多项证据，但尚不能替代人工真值、标准物质标定和最终定量积分。

## 2. 目标与边界

### 2.1 当前目标

1. 对一维色谱时间序列自动完成格式校验、去重、噪声估计、基线估计和轻度降噪。
2. 在全时间范围分别搜索正峰和负峰，输出峰顶、自动起止、宽度、面积及峰形特征。
3. 识别普通峰、未映射峰、窄峰、宽峰、重叠候选、鼓包背景上的峰、平缓宽峰、负峰、结构化背景残差和电信号干扰。
4. 对每个候选输出 `peak_confidence`，并把峰是否存在与 T1～T6 模板匹配分开解释。
5. 对疑似峰顶尖点使用多参数联合判别，避免“一个尖点覆盖整个真实峰体”，同时避免保留时间匹配救回强电瞬态。
6. 输出可逐峰复核的 CSV、逐文件质量汇总、三层检测图和不改变原始数据的分段放大图。

### 2.2 当前不承诺的能力

- 未配置组分映射时，T1～T6 只是保留时间模板槽，不代表 H2、CO、CH4 等固定化学名称。
- 尚未完成高斯、洛伦兹、EMG 等参数化拟合及重叠峰解卷积。
- 自动面积和自动边界尚未通过标准曲线、响应因子和人工积分边界校准，不能直接作为最终浓度定量结果。
- 尚无逐峰人工真值和独立盲测集，不能据当前输出计算可信的检出率、误报率或真实概率。
- 当前联合电形态模型只用于正候选；负向电瞬态仍走独立的负峰规则。

## 4. 总体处理流程

```text
原始 ZIP/CSV
  → 数值与时间轴校验
  → SHA-256 精确去重
  → 从 5 条基准谱学习 T1～T6
  → 暂定孤立脉冲识别
  → 主基线、双极性基线、rolling-ball 背景
  → 鼓包/漂移文件级与局部区间诊断
  → 随时间变化的随机噪声和弱候选有效噪声
  → 正峰检测分支与带符号负峰分支
  → SciPy find_peaks 搜索正、负候选
  → 峰顶、自动边界、宽度、面积和形态特征
  → 正候选峰体分/电干扰分与去尖反事实检查
  → 模板匹配、类型分类和峰置信度
  → confirmed/review/artifact 分级
  → CSV、统计表、三层检测图、原始分段放大图
```

## 5. 输入校验与稳健量程

程序要求时间轴有限、严格递增且至少有 5 个采样点；`x/Curvel` 转为浮点后拒绝 `NaN/Inf`。原始 `Curvel` 始终保留，后续预处理不会覆盖第一层原始数据。

用于归一化的稳健信号量程为：

```text
R = Q99.9%(raw) - Q0.1%(raw)
```

使用分位数而不是最大值减最小值，可减少单个极端毛刺对全文件阈值的支配。

## 6. 预处理技术

### 6.1 暂定孤立脉冲

原始信号先与三点中值参考比较。暂定脉冲阈值取以下较大者：

```text
8 × 残差稳健标准差
0.004 × R
```

超过阈值的孤立点只在正峰预处理分支中暂定修复；原始数组和原始图不变。该掩码只说明“相对三点中值异常”，不再一票否决整个候选峰。

### 6.2 三路基线/背景

1. 双极性稳健基线：长窗中位趋势和轻度平滑，避免强负峰把正峰基线整体下拉。
2. arPLS 基线：`λ=1e7`、最多 50 次迭代，跟随慢漂移。
3. rolling-ball 局部背景：约 `0.50 min` 半窗口和 `0.025 min` 平滑半窗口，独立描述局部鼓包。

正峰主基线取 arPLS 与双极性基线的稳健上包络；rolling-ball 不再预先与主基线合并，使两种背景模型的差异可以形成可解释的不确定度证据。

### 6.3 检测分支

- `positive`：暂定脉冲修复后减主基线，并用短窗 Savitzky–Golay 轻度平滑，用于正峰检测。
- `signed`：保留正负号的基线校正分支，主要用于负峰。

两条曲线都是检测信号，不是峰函数拟合结果。

### 6.4 动态噪声

随机噪声由短窗平滑残差局部 MAD 与一阶差分局部 MAD 的较大值形成。文件级高噪声条件为：

```text
global_noise > max(10 × reference_noise, 5e-5)
```

正向弱候选还计算：

- `structured_noise`：相对约 `0.20 min` 中值趋势的局部波纹尺度；
- `baseline_model_uncertainty`：主基线与 rolling-ball 差异的局部稳健尺度；
- `effective_noise = max(random_noise, structured_noise, baseline_model_uncertainty)`。

当 `relative_prominence < 0.005` 时，最终 SNR 使用 `effective_noise`；较强候选仍用随机噪声，避免真实峰把自身当成结构化噪声。

## 7. 鼓包、漂移和结构化背景

### 7.1 四个文件级指标

诊断基线由主基线、rolling-ball 和双极性基线的逐点中位数构成，再以约 `0.20 min` 窗口平滑。

```text
endpoint_drift_ratio = (末端窗口中位数 - 首端窗口中位数) / R

baseline_excursion_ratio =
    (max(诊断基线) - min(诊断基线)) / R

baseline_curvature =
    Q1%～Q99%(诊断基线 - 首尾连线) 的范围 / R

bump_duration = 最长可靠正向宽背景区间持续时间
```

`endpoint_drift_ratio` 可检测单向漂移，但会漏掉“先升后降”；`baseline_excursion_ratio` 和 `baseline_curvature` 正是为此补充。

### 7.2 局部黄色宽背景区

- 灵敏分支：诊断基线突出度 `>=0.005R`；
- 强分支：突出度 `>=0.03R`；
- FWHM `>=0.50 min`；
- 候选距离 `>=0.30 min`；
- 相距不超过 `0.20 min` 的区间合并。

低幅候选若峰顶原始信号泄漏超过 `0.03R`，或区间内超过该泄漏线的点多于 10%，会被拒绝，避免窄高分析峰污染宽背景判断。

`bump_strength` 分为 `none/candidate/strong`。黄色阴影表示背景区间，不表示阴影内每个候选都是假峰。

### 7.3 结构化背景残差

文件先满足任一非平稳条件：

```text
baseline_excursion_ratio >= 0.08
或 baseline_curvature >= 0.04
或 abs(endpoint_drift_ratio) >= 0.05
```

候选还需弱、密集、宽度小于 0.80 min 且双侧峰形浅。通过门控后类型为 `structured_background_residual`，参考峰宽得分减半，置信度上限为 0.74，保留在 `review` 而不删除。

## 8. 候选检测与基本特征

正、负分支分别使用 `scipy.signal.find_peaks`。每个候选输出：

- 峰顶：`apex_time_min`、`corrected_apex_value`、`raw_apex_value`；
- 自动边界：`start_time_min/end_time_min`，使用突出度基准 `rel_height=0.95`；
- 宽度：`width_min/fwhm_min` 和对应采样点宽度；
- 强度：高度、突出度、相对突出度、随机/有效 SNR；
- 形态：对称性、峰顶宽度比、面积形状、升降时间、连续性和斜率；
- 背景：局部基线变化、鼓包重叠、候选密度；
- 负峰证据：带符号下探深度、原始谱线双侧谷深。

自动面积为自动边界内基线校正正值的梯形积分，适合比较和复核，但不是最终定量面积。

## 9. 正向电干扰联合形态模型

### 9.1 固定物理时间指标

最高三个采样点占比会随采样率改变，因此当前评分改用：

```text
apex_core_area_fraction =
  integral(max(signal,0), apex-0.005, apex+0.005)
  / integral(max(signal,0), start, end)
```

积分边界插值到精确物理时间。`top3_energy_fraction` 和 `fwhm_samples` 仍输出，但只作审计，不参与评分或硬门。

### 9.2 二次拟合峰顶超额

候选与暂定脉冲掩码重合时，排除掩码点，在半宽 `max(0.010 min, 0.25×FWHM)` 内拟合周围二次峰体：

```text
apex_quadratic_excess_ratio =
  max(0, actual_apex - quadratic_prediction) / prominence
```

只有该值 `>=0.015` 才令 `apex_spike_flag=true`。该步骤防止正常窄峰仅因三点中值差而被判电尖峰。

### 9.3 峰体分

`peak_body_score` 权重：

| 证据 | 权重 |
|---|---:|
| 物理时间FWHM或模板相对宽度支持 | 14% |
| 升降斜率平衡 | 10% |
| 对称性 | 12% |
| 面积形状 | 16% |
| 非平顶性 | 8% |
| 局部基线稳定性 | 8% |
| 升降连续性 | 12% |
| 固定时间峰顶面积分散度 | 8% |
| 去尖高度/突出度/面积平均保留率 | 8% |
| 二次拟合峰顶超额较小 | 4% |

### 9.4 电干扰分

`electrical_interference_score` 权重：

| 证据 | 权重 |
|---|---:|
| 物理时间FWHM过窄 | 18% |
| 固定时间峰顶面积过度集中 | 14% |
| 归一化斜率跳变 | 14% |
| 二次拟合峰顶超额 | 10% |
| 文件级脉冲密度 | 18% |
| 低对称性 | 6% |
| 平顶异常 | 6% |
| 升降斜率失衡 | 8% |
| 去尖后峰体损失 | 6% |

### 9.5 去尖峰体硬门

先满足以下入口之一：

```text
绝对入口：fwhm_min >= 0.060 且 area_shape_factor >= 1.15
模板入口：存在模板，width_min 位于
  [0.75×模板宽度下限, 1.25×模板宽度上限]
  且 area_shape_factor >= 0.85
```

再同时满足：

```text
symmetry >= 0.75
top_width_ratio <= 0.45
baseline_change_ratio <= 0.25
min(rise_continuity, fall_continuity) >= 0.80
apex_core_area_fraction <= 0.35
despiked_height_retention >= 0.65
despiked_prominence_retention >= 0.65
despiked_area_retention >= 0.65
apex_quadratic_excess_ratio <= 0.25
```

### 9.6 路由规则

- `peak_with_apex_spike`：有可靠尖点、峰体通过硬门、峰体分 `>=0.72` 且不低于电干扰分；峰置信度乘 0.90。
- 纯电尖峰：电干扰分 `>=0.45`、峰体未通过，并且有可靠尖点或 `fwhm_min<0.018`；峰置信度乘 0.25。
- 歧义峰顶异常：有可靠尖点但既不能救回、也不足纯电尖峰；类型为电干扰候选，峰置信度乘 0.55，通常保留复核。
- 模板保留时间不能救回强电形态，只能提供相对宽度支持。

这套模型只对正候选适用。负候选 `electrical_assessment_applicable=false`，三项去尖保留率、峰体分和电干扰分在 CSV 中为空，继续走独立负峰规则。

## 10. 模板匹配、峰置信度和状态

### 10.1 模板匹配置信度

```text
template_confidence =
  50% 保留时间 + 30% 峰宽 + 15% 对称性 + 5% 非平顶性
```

它回答“像不像某个 T1～T6”，不直接决定峰是否存在。

### 10.2 峰存在置信度

正峰基础权重：SNR 25%、相对突出度 25%、参考宽度相似度 20%、对称性 15%、非平顶性 15%。

负峰基础权重：SNR 18%、相对突出度 15%、宽度相似度 12%、对称性 10%、带符号下探 20%、原始双侧谷深 25%。

高噪声弱事件乘 0.75，靠近采集边缘乘 0.65；其他形态乘数按类型施加。`peak_confidence` 是可解释综合分，不是严格校准概率。

### 10.3 状态

| 状态 | 规则 |
|---|---|
| confirmed | `peak_confidence >= 0.75` |
| artifact | 置信度 `<0.45`，且类型为电事件或峰/噪声 |
| review | 其余未确认但仍需保留的事件 |

## 13. 代表性歧义案例

### 13.1 A7：尖顶不等于纯电尖峰

| 峰顶/min | 类型/状态 | peak confidence | body score | electrical score | apex core | quadratic excess | 去尖高度保留 |
|---:|---|---:|---:|---:|---:|---:|---:|
| 0.433333 | peak_with_apex_spike / confirmed | 0.82494 | 0.9553 | 0.0844 | 0.1278 | 0.0525 | 0.9951 |
| 0.565000 | peak_with_apex_spike / confirmed | 0.89969 | 0.9606 | 0.0552 | 0.0754 | 0.0400 | 0.9978 |

二次拟合确认峰顶有额外尖点，但去尖后完整峰体几乎全部保留，因此判为“真实峰带峰顶尖点”。两峰自动窗口重叠的信息保留在 `reasons`，不会覆盖该质量类别。

### 13.2 H2 0.600 min：高峰体分仍需硬门

该事件为 `electrical_interference_candidate/review`，峰置信度 0.51279、峰体分 0.8685、电干扰分 0.0485。其对称性、连续性和去尖保留率都高，但没有模板相对宽度入口，`area_shape_factor=0.9852` 又低于绝对入口 1.15，故不能自动救回；同时电干扰分不足 0.45，不能直接判纯电尖峰。保留 review 是对证据冲突的显式表达。

### 13.3 A3：首尾接近仍是鼓包

```text
endpoint_drift_ratio ≈ -0.040
baseline_excursion_ratio ≈ 0.268
baseline_curvature ≈ 0.277
bump_duration ≈ 6.52 min
```

首尾差会抵消，但全程起伏、非线性弯曲和持续时间能识别中段鼓包。

### 13.4 绿色窗口与黄色阴影

绿色窗口是模板容差，不表示检测到峰；黄色阴影是宽背景，不表示整段是假峰。二者都必须与峰顶状态符号和 CSV 行结合解释。

## 14. 验证体系

当前测试文件定义 20 项回归检查，覆盖：

- 6 个参考模板槽和五份参考谱 T1/T2；
- F1 强正峰/负峰；
- H1/H2/H3/A4 电干扰保护；
- A7 带尖点真实峰体；
- 与 T1 保留时间重合的合成毛刺；
- 正常窄 T1、叠加单点尖刺的 T1；
- 多采样率高斯峰一致性；
- B10 高噪声不整文件拒绝；
- A1/A2/A3/A8/A10/A12/A14 鼓包目标；
- 结构化背景不能自确认；
- 确认峰边界和峰顶标注；
- 原始分段图等时分段且不修改输入数据。

这些是回归保护而非真实准确率评测。下一阶段必须建立人工逐峰真值、盲测划分和版本间固定对比集。

## 16. 尚未闭环的能力

1. 逐峰人工真值：缺少人工峰顶、边界、类型和组分标签。
2. 置信度校准：当前分数尚未通过 ROC/PR、可靠性曲线或概率校准验证。
3. 平缓峰真值：保守规则仍可能低召回，需多尺度低频分支和人工标签。
4. 鼓包定量验证：缺少文件级鼓包真值和阴影区逐候选真值。
5. 重叠峰解卷积：尚无多峰参数拟合、面积拆分和不确定度。
6. 正负重叠定量：尚未完成正负前/后/中间关系的系统分类和面积分配。
7. 负向电形态模型：当前正向联合模型不直接适用于负候选。
8. 组分映射：T1～T6 尚未配置实际化学组分。
9. 最终定量：缺少标准曲线、响应因子、人工积分边界和仪器间校准。
10. C++/ARM：需在算法和真值稳定后再移植，避免固化尚未验证的经验阈值。

## 17. 后续研究与扩展建议

### 17.1 优先建立真值

- 逐文件标记工况：正常、鼓包、漂移、高噪声、电干扰。
- 逐候选标记：是否真峰、方向、类型、组分、峰顶、起止、是否叠加尖点。
- 固定训练/校准/盲测划分，保留仪器型号和采样率分层。

### 17.2 校准两类置信度

- 对 `peak_confidence` 输出 ROC、PR、召回率、误报率和可靠性图。
- 分别校准普通峰、负峰、鼓包峰、电干扰和高噪声场景。
- 将 `peak_body_score` 与 `electrical_interference_score` 作为二维证据，而不是只调一个阈值。

### 17.3 继续验证采样率不变性

- 对同一解析峰在多种采样率下重采样。
- 检查物理时间 FWHM、固定时间峰顶面积、二次拟合超额和最终类别的一致性。
- `fwhm_samples/top3_energy_fraction` 只用于发现采样不足，不重新进入主评分。

### 17.4 重叠峰与面积

- 对已标记重叠区域尝试高斯/EMG 混合模型和非负最小二乘。
- 用残差、AIC/BIC、参数置信区间判断是否真的需要多峰。
- 面积分配前先由人工积分边界和标准物质校准。

### 17.5 负峰扩展

- 建立负向毛刺、独立负峰、正峰前后负扰动和峰间谷底的专门真值集。
- 研究镜像的固定时间峰顶面积和二次拟合谷底超额，但不要直接复制正峰阈值。

### 17.6 参数与版本治理

- 将所有阈值和权重写入版本化配置和 manifest。
- 每次修改都对固定盲测集输出状态迁移表。
- 保留原始 CSV 哈希、代码版本、配置版本和输出目录，确保可复现。
