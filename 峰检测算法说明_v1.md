# 色谱峰检测与类型判定算法 v0.2

## 1. 当前交付结论

本版本已经按“先预处理、再找峰、最后判类型”的顺序跑完所给数据。输入共 45 个 CSV，识别到 4 组内容完全相同的重复文件，实际处理 41 条独立曲线。算法从 5 条基准谱自动学习出 6 个稳定正峰模板；5 条基准谱的 6 个模板峰均被确认检出。

当前输出的是峰的形态类型和位置，不会在缺少映射依据时猜测 H2、CO、CH4、CO2 等化学组分。组分名称需要在 `component_map.json` 中补齐 T1～T6 的对应关系。

由于异常数据没有逐峰人工真值，不能把“本次运行结果”直接宣称为最终正确答案。本版本采用保守策略：证据充分的正峰标为“确认”；负峰、宽峰、高噪声及有歧义事件标为“待复核”；明显电尖峰/电干扰标为“干扰/伪峰”。这正适合下一步逐条人工验证和迭代阈值。

## 2. 算法处理流程

1. 数据检查：读取 `x,Curvel`，按内容哈希去除重复曲线，保留原文件来源。
2. 孤立脉冲预处理：仅替换达到强异常阈值的单点脉冲；较宽事件不直接删除，留给后面的干扰分类。
3. 基线校正：arPLS 负责慢漂移，双极性中位数基线防止负峰把 arPLS 长时间向下拖拽，二者取稳健上包络作为正峰主基线；rolling-ball 作为局部鼓包背景的辅助证据。
4. 降噪：短窗 Savitzky–Golay 平滑；由平滑残差和局部一阶差分 MAD 共同估计随时间变化的噪声。
5. 候选检测：SciPy `find_peaks` 分别检测正峰和负峰，计算峰顶、起止、95% 峰宽、半高宽、突出度、局部 SNR、面积、对称性和相对幅度。
6. 模板学习：从 5 条基准谱学习 6 个峰的保留时间、允许漂移、宽度和对称性范围。
7. 类型判定：综合保留时间、峰宽、对称性、突出度、噪声工况、局部背景变化、相邻峰窗口和脉冲标记。
8. 可靠性分级：`confirmed/确认`、`review/待复核`、`artifact/干扰或伪峰`。

## 3. 对 5.1～5.6 的对应

| 文档问题 | 当前算法处理 |
|---|---|
| 5.1 动态噪声 | 局部 MAD 噪声、局部 SNR、整条曲线高噪声保护；高噪声时提高相对突出度门槛。 |
| 5.2 基线漂移 | arPLS + 双极性稳健基线；输出漂移量、漂移占量程比例和漂移标记。 |
| 5.3 平缓宽峰与基线 | 输出“宽鼓包、宽峰或基线”，不强行把无法区分的宽事件确认为组分峰。 |
| 5.4 重叠峰/大峰上的小峰 | 用 95% 峰窗口交叠、模板峰宽包络和 rolling-ball 局部背景变化，输出“重叠正峰”“宽峰或未分离重叠峰”“鼓包或漂移背景上的正峰”。 |
| 5.5 电信号干扰 | 用单点脉冲、异常窄峰、峰顶平坦度、对称性和模板宽度不匹配判定。H3 的约 4.363 min 平顶事件已判为电信号干扰候选。 |
| 5.6 负峰 | 独立双极性分支；要求原始曲线呈双侧谷形且相对基线确实下探。峰间浅谷会标为“峰间谷底或负峰”，避免强行确认。 |

## 5. 输出文件怎么用

- `reliable_detector_results/detected_peaks_for_validation.csv`：人工逐峰检查的主表，包含确认和待复核事件。
- `reliable_detector_results/confirmed_peaks.csv`：算法确认的正峰。
- `reliable_detector_results/review_required.csv`：负峰、宽峰、重叠、鼓包及噪声歧义事件。
- `reliable_detector_results/interference_candidates.csv`：电尖峰和电信号干扰候选。
- `reliable_detector_results/priority_validation_set.csv`：建议优先人工核验的集合，已排除纯“峰或噪声”低优先级项。
- `reliable_detector_results/validation_labels_template.csv`：可直接填写人工真值的完整模板，末尾预留人工位置、类型、组分和备注列。
- `reliable_detector_results/all_detected_features.csv`：全部输出及数值特征。
- `reliable_detector_results/file_quality_summary.csv`：逐文件的噪声、漂移、峰数和异常数。
- `reliable_detector_results/learned_peak_template.json`：从基准谱学习的 T1～T6 模板。
- `reliable_detector_results/plots/`：每条独立曲线的原始信号、两种背景和检测标记图。

主表关键字段：

- `apex_time_min/start_time_min/end_time_min`：峰顶和边界位置，单位 min。
- `feature_type_cn`：中文峰类型。
- `status_cn`：确认、待复核、干扰/伪峰。
- `template_slot/component`：模板槽和可选组分名称。
- `width_min/fwhm_min/prominence/snr/symmetry/area`：形态与强度证据。
- `relative_prominence`：相对本文件量程的突出度。
- `confidence/reasons`：置信度和判定原因。

## 6. 运行方式

当前目录已包含输入数据的工作副本，直接运行：

```powershell
python reliable_peak_detector.py
```

也可以指定原始压缩包和新输出目录：

```powershell
python reliable_peak_detector.py --zip "D:\色谱峰处理算法\一些典型的谱图.zip" --out reliable_detector_results
```

缺少依赖时：

```powershell
python -m pip install -r requirements-peak-detector.txt
```
